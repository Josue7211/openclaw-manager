# Notes Parity Roadmap

This is the source-of-truth checklist for the long-running notes goal: reach practical parity with Google Docs and Obsidian while keeping ClawControl local-first and sync-aware.

Parity is not done until every required row is either implemented and verified, intentionally superseded by a better ClawControl-native workflow, or explicitly rejected with product rationale. UI polish alone is not parity.

## Status Legend

- `done`: implemented and covered by tests or manual runtime verification.
- `partial`: meaningful implementation exists, but behavior, UI, sync, permissions, import/export, or verification is incomplete.
- `missing`: no usable implementation.
- `unknown`: needs audit before status can be trusted.
