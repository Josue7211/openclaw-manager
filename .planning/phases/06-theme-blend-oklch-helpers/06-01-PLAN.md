    1. `hexToOklch(hex: string): [number, number, number]` -- returns [L, C, H] where L is 0-1, C is 0-0.4ish, H is 0-360.
       Conversion pipeline: hex -> sRGB (0-1) -> linear RGB (gamma decode) -> OKLab (via M1 matrix) -> OKLCH (polar).
       Use the Bjorn Ottosson OKLab matrices:
       - sRGB to linear: `c <= 0.04045 ? c / 12.92 : ((c + 0.055) / 1.055) ** 2.4`
       - Linear RGB to LMS (cube root): apply M1 matrix, then cube root each
       - LMS to OKLab: apply M2 matrix
       - OKLab to OKLCH: L = L, C = sqrt(a*a + b*b), H = atan2(b, a) * 180/PI (normalize to 0-360)

    2. `oklchToHex(oklch: [number, number, number]): string` -- returns 7-char hex.
       Reverse pipeline: OKLCH -> OKLab -> LMS (cube) -> linear RGB (M1 inverse) -> sRGB (gamma encode) -> hex.
       - OKLCH to OKLab: a = C * cos(H * PI/180), b = C * sin(H * PI/180)
       - OKLab to LMS: apply M2 inverse, then cube each
       - LMS to linear RGB: apply M1 inverse
       - Linear to sRGB: `c <= 0.0031308 ? 12.92 * c : 1.055 * c**(1/2.4) - 0.055`
       - Clamp each channel to 0-255, format as hex

    3. `interpolateHexOklch(hex1: string, hex2: string, t: number): string`
       - Clamp t to [0, 1]
       - Convert both to OKLCH
       - Interpolate L, C linearly: `v1 + (v2 - v1) * t`
       - Interpolate H using shortest-arc hue interpolation (handle 350->10 wrapping)
       - Handle achromatic colors (C near zero): if either color is achromatic (C < 0.002), use the other's hue; if both achromatic, hue = 0
       - Convert result back to hex
