Create four sub-components for the Sessions page. All must follow accessibility rules (buttons not divs, aria-labels on icon-only buttons, aria-live for dynamic content).

**1. `SessionCard.tsx`** -- React.memo card component:

- Props: `{ session: ClaudeSession; selected: boolean; onSelect: () => void; onKill: (id: string) => void; available: boolean; isKilling: boolean }`
- Use `<button type="button">` as the outer element with `aria-pressed={selected}`
- Status dot with pulse animation when `status === 'running'` (use `STATUS_COLORS` from types.ts)
- Status badge text from `STATUS_LABELS`
- Show session.task as title (truncate with text-overflow: ellipsis)
- Show session.model in monospace badge (like AgentCard model badge)
- Show duration: if `session.duration` is set, format as `Xm Ys`. If null but `startedAt` is set, show a live-updating timer using `useEffect` + `setInterval(1000)`. If neither, show nothing.
- Kill button (X icon) on the card -- only shown when status is 'running' or 'paused'. Uses inline confirmation pattern:
  - First click sets `confirmingKill` state
  - Shows "Kill?" text button, second click calls `onKill(session.id)`
  - 3-second timeout resets confirmingKill to false
  - Disabled when `!available` (same opacity/cursor pattern as AgentDetailPanel)
- Styles: match AgentCard -- bg-card background, accent border when selected, 16px borderRadius, backdrop-filter blur

**2. `SessionList.tsx`** -- Scrollable list with React Query data fetching:

- Uses `useQuery` with `queryKeys.claudeSessions` and `refetchInterval: 5000`
- Uses `useMutation` for create and kill with `onSuccess: () => queryClient.invalidateQueries({ queryKey: queryKeys.claudeSessions })`
- Props: `{ selectedId: string | null; onSelect: (id: string) => void }`
- Header bar with "Sessions" title and "New Session" button (Plus icon from @phosphor-icons/react)
- Clicking "New Session" toggles `showForm` state
- When `showForm` is true, renders `<NewSessionForm>` inline below the header
- Maps sessions to `<SessionCard>` components
- Empty state when no sessions: centered text "No active sessions" with muted color
- Loading state: centered "Loading..." text
- Error state when `available === false`: show "OpenClaw is unreachable" banner at top of list

**3. `NewSessionForm.tsx`** -- Inline form for spawning sessions:

- Props: `{ onSubmit: (payload: CreateSessionPayload) => void; onCancel: () => void; isSubmitting: boolean; available: boolean }`
- Task textarea (required, placeholder "Describe the task...", max 2000 chars to match backend validation)
- Working directory input (optional, placeholder "/path/to/project")
- Model text input (optional, placeholder "e.g. opus, sonnet")
- Submit button "Start Session" -- disabled when task is empty, when isSubmitting, or when !available
- Cancel button to close the form
- All inputs have `aria-label` attributes
- Style: card-like background with border, 12px padding, 12px borderRadius

**4. `SessionOutputPanel.tsx`** -- Right panel with xterm.js output viewer:

- Props: `{ sessionId: string | null }`
- When `sessionId` is null: show empty state "Select a session to view output" centered with muted text
- When `sessionId` is set:
  - Header bar showing the session ID (or a short version)
  - Connection status indicator: green dot + "Connected" when connected, amber dot + "Connecting..." when not
  - `useSessionOutput(containerRef, sessionId)` hook invocation
  - Terminal container `<div ref={containerRef} className="terminal-container">` with flex: 1 to fill remaining space
  - If `error` is set, show error banner above the terminal
- **Key:** Use `key={sessionId}` on the component wrapper to force unmount/remount when session changes -- this ensures clean terminal disposal and WebSocket reconnection per Pitfall 2 from research
