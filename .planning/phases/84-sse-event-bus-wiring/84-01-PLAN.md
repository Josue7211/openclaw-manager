<objective>
Wire the SSE event bus to actual gateway WebSocket events so the frontend receives
real-time gateway events (agent status, chat, presence, cron, etc.) instead of
relying on polling.

Purpose: This is the foundation for phases 85 (agent event streaming) and 86
(session event streaming). Without this SSE bridge, the frontend has no way to
receive push events from the gateway.

Output: A new `/api/gateway/events` SSE endpoint, a `/api/gateway/status` REST
endpoint, and a `useGatewaySSE` frontend hook that connects them.
</objective>
