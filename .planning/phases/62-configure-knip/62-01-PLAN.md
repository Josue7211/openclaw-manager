**Why this configuration:**

- **entry**: Every file that is a dynamic import target (React.lazy, widget registry `component: () => import(...)`, bjorn-store `exposePrimitivesAPI`, primitives register.ts). Without these, knip sees no static import chain to these files and flags them as unused. The list was built by exhaustively scanning all `lazy(() => import(...))` calls and all `component:` factory functions in widget-registry.ts.

- **project**: All TypeScript/TSX source files in src/ are candidates for analysis.

- **ignore**: Test files are not production entry points. `vite-env.d.ts` is auto-generated.

- **ignoreDependencies**: Tauri packages are externalized at build time (see vite.config.ts `rollupOptions.external`) so knip won't find static imports for them. Type definition packages (`@types/*`) are used by the compiler only. Dev tool packages (eslint plugins, prettier, vitest, etc.) are consumed via config files, not imports.
