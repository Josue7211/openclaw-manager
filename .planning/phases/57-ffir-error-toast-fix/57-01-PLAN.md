    This is a diagnostic-then-fix task. The string "ffir" does NOT exist in source code -- the root cause must be found at runtime.

    **Step 1: Remove stale sidecar binaries**

    The directory `src-tauri/binaries/` contains stale node sidecar files left over from before the migration to embedded Axum (main.rs line 232: "replaces Node.js sidecar"). These files are:
    - `node-` (symlink to /usr/bin/node -- invalid sidecar name)
    - `node-~` (symlink to /usr/bin/node -- invalid sidecar name)
    - `node-x86_64-unknown-linux-gnu` (63MB ELF binary)

    Tauri's build system scans `binaries/` for sidecar executables matching the pattern `name-{target_triple}`. Even without an `externalBin` entry in tauri.conf.json, the build process or dev server may attempt to register these, and the malformed names `node-` and `node-~` could produce garbled error messages.

    Delete all three files. Keep only `.gitkeep` in the directory.

    **Step 2: Clean Rust build artifacts**

    Run from project root:
    ```bash
    cd src-tauri && cargo clean -p openclaw
    ```
    Note: The package name is "openclaw" (Cargo.toml line 2), not "mission-control".

    **Step 3: Add temporary debug interceptors**

    In `frontend/src/main.tsx`, add BEFORE the `runMigrations()` call (around line 81), immediately after all imports:

    ```typescript
    // --- TEMPORARY: ffir error diagnostic (remove after root cause identified) ---
    window.addEventListener('unhandledrejection', (event) => {
      const reason = event.reason
      const msg = reason?.message || String(reason)
      if (msg.includes('ffir') || msg.includes('Executable') || msg.includes('not found')) {
        console.error('[FFIR DEBUG] Unhandled rejection:', reason)
        console.error('[FFIR DEBUG] Stack:', reason?.stack)
        event.preventDefault()
      }
    })
    // --- END TEMPORARY ---
    ```

    **Step 4: Rebuild and test**

    Run `cargo tauri dev` from project root. Load any page. Watch terminal and browser devtools for `[FFIR DEBUG]` output. The stack trace reveals the exact source.

    If the error no longer appears after Step 1+2 (removing binaries + clean build), that confirms stale sidecar files were the root cause. Proceed to Step 5.

    If the error still appears, the stack trace from Step 3 will point to the source. Apply the appropriate fix:
    - Tauri plugin init failure: wrap plugin init in main.rs or remove unused plugin
    - Missing Rust crate for frontend plugin: remove unused npm dep or add crate
    - WebKitGTK native overlay: add permanent suppression guard

    **Step 5: Remove temporary debug code and add permanent guard**

    Remove the temporary diagnostic block. Replace with a permanent, minimal guard. Add this BEFORE the `runMigrations()` call, inside a `__TAURI_INTERNALS__` check:

    ```typescript
    // Prevent Tauri plugin/IPC errors from surfacing as WebKitGTK error overlays.
    // All Tauri IPC calls below have explicit .catch() handlers; this catches
    // any errors that escape those handlers (e.g., plugin init failures,
    // missing optional binaries).
    if (window.__TAURI_INTERNALS__) {
      window.addEventListener('unhandledrejection', (event) => {
        const msg = event.reason?.message || String(event.reason)
        if (msg.includes('Executable') || msg.includes('not found') || msg.includes('plugin')) {
          console.debug('[Tauri] Suppressed non-critical error:', msg)
          event.preventDefault()
        }
      })
    }
    ```

    This guard is acceptable because:
    1. Only fires in Tauri mode (browser mode unaffected)
    2. Only suppresses specific error patterns (missing binaries/plugins)
    3. Logs at debug level for diagnosis
    4. All actionable errors still propagate normally
    5. Does NOT suppress the error itself -- just prevents WebKitGTK from showing an overlay

    **Step 6: Verify fix**

    After applying the fix, run `cargo tauri dev` again and confirm:
    - No error toast/overlay appears on page load
    - No `[FFIR DEBUG]` or similar in console
    - All pages load cleanly
