<objective>
Fix browser-mode authentication so developers can run `npm run dev` and authenticate against the Axum backend without needing the Tauri shell.

Purpose: Currently, AuthGuard auto-skips auth in browser dev mode (`devNoBackend` bypass), which means no real session is established and all `RequireAuth` API calls fail with 401. The OAuth callback also strands browser users on port 3000 with a "close this tab" page instead of redirecting them back to the frontend on port 5173.

Output: Working browser-mode auth flow -- AuthGuard sends to login, OAuth redirects back to frontend, demo mode preserved.
</objective>
