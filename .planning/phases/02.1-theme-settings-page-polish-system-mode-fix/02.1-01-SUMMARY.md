## Decisions Made
- gsettings `color-scheme` checked first (GNOME 42+ and many Wayland compositors set this), with `gtk-theme` name as secondary fallback
- Non-Linux platforms return `false` from the Rust command since Tauri native detection works correctly on macOS/Windows
- The `onThemeChanged` listener was left unchanged -- on Linux, GTK theme changes at runtime are rare, and the startup check is sufficient
- Dynamic import of `@tauri-apps/api/core` in the fallback path keeps bundle size unchanged for non-Linux platforms
