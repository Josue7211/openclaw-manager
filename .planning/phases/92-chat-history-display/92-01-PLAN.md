1. Add imports at the top of `gateway.rs`:
   - `use axum::extract::Path;` (for `:key` extraction)
   - `use serde::Deserialize;` (for query params struct)

2. Add query params struct above the handler:
```rust
#[derive(Debug, Deserialize)]
struct HistoryQueryParams {
    limit: Option<u32>,
}
```

3. Add the handler function (place it before the `router()` function). CRITICAL: Do NOT use `gateway_forward` — it rejects `?` in paths. Use `state.http` directly, following the same pattern as `fetch_remote_history` in `chat.rs`.

Handler signature:
```rust
async fn gateway_session_history(
    State(state): State<AppState>,
    RequireAuth(_session): RequireAuth,
    Path(key): Path<String>,
    axum::extract::Query(params): axum::extract::Query<HistoryQueryParams>,
) -> Result<Json<Value>, AppError>
```

Handler logic:
- Validate key: reject empty or >100 chars with `AppError::BadRequest("invalid session key")`
- Percent-encode the key using `crate::routes::util::percent_encode(&key)`
- Get base URL via `openclaw_api_url(&state)`, return BadRequest if not configured
- Get API key via `openclaw_api_key(&state)`
- Clamp limit: `params.limit.unwrap_or(50).min(200)`
- Build URL: `"{base}/chat/history/{encoded_key}?limit={limit}"`
- Build request with `state.http.get(&url).header("Content-Type", "application/json").timeout(Duration::from_secs(30))`
- Add `Authorization: Bearer {api_key}` header if key is not empty
- Send request, handle errors: 4xx -> BadRequest with `sanitize_error_body`, 5xx -> Internal
- Parse response as `Value` and return `Json(value)`

4. Register the route in `router()`:
```rust
.route("/gateway/sessions/:key/history", get(gateway_session_history))
```
