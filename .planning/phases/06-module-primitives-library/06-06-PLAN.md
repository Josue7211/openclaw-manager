<task type="auto" tdd="true">
  <name>Task 1: Build TimerCountdown primitive</name>
  <files>
    frontend/src/components/primitives/TimerCountdown.tsx,
    frontend/src/components/primitives/__tests__/TimerCountdown.test.tsx,
    frontend/src/components/primitives/register.ts
  </files>
  <behavior>
    - Counts down from config.duration seconds (default 60) to 0, OR counts up from 0 based on config.direction ("down"/"up")
    - Displays time in MM:SS format (or HH:MM:SS if > 3600 seconds)
    - Optional milliseconds display when config.showMilliseconds is true (MM:SS.ms)
    - Start/pause button toggles the timer
    - Reset button returns to initial state
    - Auto-start when config.autoStart is true
    - When countdown reaches 0, timer stops and displays "00:00" with accent color flash
    - Timer interval properly cleaned up in useEffect return (Pitfall 3 from research)
    - Uses useRef for interval ID to prevent stale closure issues
    - Exports configSchema with: duration (number, min 1, max 86400, default 60), direction (select: down/up), autoStart (toggle, default false), showMilliseconds (toggle, default false)
  </behavior>
  <action>
    **TimerCountdown.tsx:**
    - React.memo wrapped, implements WidgetProps
    - Co-exports configSchema with 4 fields: duration, direction, autoStart, showMilliseconds
    - State: useState for elapsed (milliseconds), useState for isRunning (boolean)
    - Refs: useRef for interval ID (number | null), useRef for lastTick timestamp
    - Timer logic: Use setInterval(callback, showMilliseconds ? 50 : 1000). In callback, compute delta from Date.now() - lastTick.current. Update elapsed. For countdown: if elapsed >= duration * 1000, stop and clamp.
    - useEffect cleanup: return () => { if (intervalRef.current) clearInterval(intervalRef.current) } -- CRITICAL for preventing memory leaks
    - useEffect for autoStart: if autoStart and not running, start on mount
    - Display computation:
      - For countdown: remaining = Math.max(0, duration * 1000 - elapsed). Format remaining.
      - For countup: format elapsed directly.
      - MM:SS format: minutes = Math.floor(ms / 60000), seconds = Math.floor((ms % 60000) / 1000)
      - HH:MM:SS if total > 3600s: hours = Math.floor(ms / 3600000)
      - Milliseconds: .XX (two digits) appended
    - Layout: centered flex column. Time display large (48px, font-weight 700, font-variant-numeric tabular-nums, color var(--text-primary)). Countdown-complete state: color var(--accent).
    - Controls: flex row of buttons below time. Play/Pause button (Play/Pause icons from Phosphor), Reset button (ArrowCounterClockwise icon). Styled: 36px circular buttons, hover-bg class, var(--text-secondary) color.
    - Label from config.title if provided, shown above time display

    **register.ts:**
    - Add registerWidget for prim-timer: category 'primitives', defaultSize {w:2, h:2}, minSize {w:2, h:2}, icon 'Timer'

    **Tests:**
    - Renders initial time display (e.g., "01:00" for 60s countdown)
    - Play button starts the timer (use vi.useFakeTimers)
    - Pause button stops the timer
    - Reset button returns to initial state
    - Countdown reaches 0 and stops
    - Countup mode increments from 0
    - Interval cleaned up on unmount (spy on clearInterval)
  </action>

    - Displays HH:MM:SS or MM:SS.ms formats as configured

  </done>
