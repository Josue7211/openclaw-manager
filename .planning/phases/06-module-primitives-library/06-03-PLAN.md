  <behavior>
    - Renders SVG polyline from config.data (number[]) with normalized coordinates
    - When config.data has fewer than 2 points, shows EmptyState with ChartLine icon
    - Renders optional grid lines when config.showGrid is true (default true)
    - Renders optional data point dots when config.showDots is true (default false)
    - Renders x-axis labels from config.labels (string[]) if provided
    - Renders y-axis labels (auto-computed from min/max of data, 3-5 tick marks)
    - Line color resolves from config.lineColor via resolveColor()
    - Tooltip on hover shows data value at that point (positioned via CSS transform, not portal)
    - Title from config.title rendered above chart area
    - Exports configSchema with fields: title (text), lineColor (select), showGrid (toggle), showDots (toggle)
  </behavior>
  <action>
    **LineChart.tsx:**
    - React.memo wrapped, implements WidgetProps
    - Co-exports configSchema: WidgetConfigSchema with fields: title (text, default "Line Chart"), lineColor (select: accent/secondary/tertiary/red/amber, default "accent"), showGrid (toggle, default true), showDots (toggle, default false)
    - SVG Layout: Use a coordinate system with padding for axes. Chart area inside the SVG viewBox: left padding 40px for y-labels, bottom padding 24px for x-labels, top padding 8px, right padding 8px. Total viewBox: "0 0 WIDTH HEIGHT" where WIDTH and HEIGHT are computed from the container.
    - Use percentage-based container (width:100%, height:100%) and a fixed viewBox (e.g., "0 0 400 200"). This avoids ResizeObserver complexity.
    - Data normalization: map each data point to chart-area coordinates. X: evenly spaced across chart width. Y: linear interpolation between min and max values to chart height (inverted -- SVG y=0 is top).
    - Grid: horizontal dashed lines at y-axis tick positions, using stroke="var(--border)", stroke-dasharray="4 4", opacity 0.5
    - Axis labels: SVG text elements. Y-axis: 3-5 rounded tick values. X-axis: from config.labels or point indices.
    - Line: SVG polyline with points string, fill="none", stroke={resolveColor(lineColor)}, strokeWidth=2, strokeLinecap="round", strokeLinejoin="round"
    - Dots: SVG circle at each data point, r=3, fill={resolveColor(lineColor)}, only when showDots is true
    - Tooltip: Use React useState for hovered point index. On SVG mousemove, compute nearest point. Render a div positioned with CSS transform relative to the chart container (not a portal). Show point value and label. Style: background var(--bg-card-solid), border var(--border), border-radius var(--radius-sm), small shadow, font-size 12px.
    - All text in SVG uses fill="var(--text-muted)" for axis labels, fill="var(--text-primary)" for title

    **register.ts:**
    - Add registerWidget for prim-line-chart: category 'primitives', defaultSize {w:4, h:3}, minSize {w:3, h:2}, icon 'ChartLine'

    **Tests:**
    - Renders SVG element with polyline when data provided
    - Shows EmptyState when data is empty array or has < 2 points
    - Grid lines present when showGrid is true
    - Grid lines absent when showGrid is false
    - Dot circles present when showDots is true
    - Title text rendered

  <behavior>
    - Renders SVG rect elements for each data value
    - Supports config.orientation: "vertical" (default) and "horizontal"
    - Supports config.stacked toggle for stacked bars (multiple series)
    - config.data: number[] for single series, or number[][] for multi-series
    - config.labels: string[] for category labels on the category axis
    - config.colors: string[] for multi-series bar colors (resolved via resolveColor)
    - Shows EmptyState with ChartBar icon when data is empty
    - Renders y-axis value labels (vertical) or x-axis value labels (horizontal)
    - Exports configSchema with fields: title, orientation (select), stacked (toggle), barColor (select)
  </behavior>
  <action>
    **BarChart.tsx:**
    - React.memo wrapped, implements WidgetProps
    - Co-exports configSchema: WidgetConfigSchema with fields: title (text, default "Bar Chart"), orientation (select: vertical/horizontal, default "vertical"), stacked (toggle, default false), barColor (select: accent/secondary/tertiary/red/amber, default "accent")
    - SVG Layout: Same viewBox approach as LineChart ("0 0 400 200"), padding for axis labels
    - Single series (config.data is number[]): Compute bar width = chartWidth / data.length with gap (80% bar, 20% gap). Bar height = (value / maxValue) * chartHeight. Position: x = index * (barWidth + gap), y = chartHeight - barHeight (vertical).
    - Multi-series (config.data is number[][]): Grouped bars side-by-side within each category slot. Colors from config.colors array, each resolved via resolveColor. If config.colors not provided, cycle through ["accent", "secondary", "tertiary"].
    - Stacked mode: When stacked=true and multi-series, stack bars vertically. Each series starts where the previous ended. Y position = chartHeight - cumulative height.
    - Horizontal mode: Swap x/y axes. Bars grow left-to-right. Category labels on y-axis, value labels on x-axis.
    - Category labels: SVG text at each bar position (config.labels or index numbers)
    - Value axis labels: 3-5 auto-computed tick marks with rounded values
    - Tooltip on hover: Show exact value on bar hover (CSS transform positioned, same pattern as LineChart)
    - Bar corners: rx=2 for subtle rounding
    - All colors from resolveColor/CSS variables

    **register.ts:**
    - Add registerWidget for prim-bar-chart: category 'primitives', defaultSize {w:4, h:3}, minSize {w:3, h:2}, icon 'ChartBar'

    - Shows EmptyState when data is empty
    - Vertical orientation: bars grow upward
    - Horizontal orientation: bars grow rightward (test rect attributes)
    - Multi-series renders multiple bars per category
    - Title text rendered
