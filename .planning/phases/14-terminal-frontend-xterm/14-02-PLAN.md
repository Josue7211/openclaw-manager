**Frontend (useTerminal.ts):**

1. Add `import { api } from '@/lib/api'` at the top

2. Inside the main useEffect (after `mountedRef.current = true` but BEFORE creating the Terminal instance), add a pre-flight capacity check:
   ```typescript
   // Pre-flight: check terminal capacity before creating anything
   let capacityOk = true
   try {
     const status = await api.get<{ active: number; max: number; available: number }>('/api/terminal/status')
     if (status.available <= 0) {
       setError(`Too many terminal sessions (max ${status.max})`)
       capacityOk = false
     }
   } catch {
     // If status check fails, proceed anyway — the WebSocket upgrade
     // will fail with its own error. Don't block on a status check failure.
   }
   if (!capacityOk || !mountedRef.current) return
   ```

3. Since the useEffect now needs to be async internally, wrap the body in an immediately-invoked async function:
   ```typescript
   useEffect(() => {
     const container = containerRef.current
     if (!container) return

     mountedRef.current = true
     let term: Terminal | null = null
     let ws: WebSocket | null = null
     // ...other refs for cleanup

     const setup = async () => {
       // Pre-flight check
       // ...existing Terminal creation + WebSocket code...
     }

     setup()

     return () => {
       // cleanup uses the local variables
     }
   }, [])
   ```

4. Restructure the cleanup function to use local variables captured by the closure (term, ws, resizeObserver, themeObserver, disposables) so cleanup works correctly with the async setup pattern. The refs (termRef, fitAddonRef, wsRef) should still be set for the font size effect.

5. Also improve the `ws.onclose` handler to detect immediate rejection (when the WebSocket closes without ever opening, which is the current behavior when the backend rejects with 429):
   ```typescript
   ws.onclose = (event) => {
     if (!mountedRef.current) return
     setConnected(false)
     // If WebSocket closed without ever connecting (code 1006 = abnormal),
     // and no error was already set by the pre-flight check, show a generic error
     if (!connected && !error && event.code === 1006) {
       setError('Terminal connection failed')
     }
   }
   ```
   Note: The `connected` and `error` checks here should use refs or local variables to avoid stale closure issues. Use a local `didOpen` boolean flag set in onopen instead.

**Key constraint:** Do NOT change the WebSocket upgrade handler or the CAS guard logic in terminal.rs. The pre-flight check is additive — it does not replace the backend's 429 rejection. If two widgets race past the pre-flight check, one will still get a WebSocket rejection, and the improved onclose handler will show a generic error.
