    2. **`sanitize_error_body(body: &str) -> String`** (pub(crate)):
       - Use `crate::redact::redact()` as the first pass to strip API keys, tokens, JWTs, hex strings.
       - Then apply additional gateway-specific sanitization:
         - Replace Tailscale/internal IPs matching `\b100\.\d{1,3}\.\d{1,3}\.\d{1,3}(:\d+)?\b` with `[redacted-host]`
         - Replace Unix file paths matching `/[a-zA-Z0-9_./-]{10,}` with `[redacted-path]`
         - If body contains "stack trace", "panic", "at module.exports", or "Traceback" -- truncate to first line only
       - Return the sanitized string.

    3. **`validate_gateway_path(path: &str) -> Result<(), AppError>`** (pub(crate)):
       - Reject paths containing ".." (directory traversal)
       - Reject paths containing "?" or "&" (query string injection -- query params should be passed via the body or explicit parameters, not smuggled in the path)
       - Reject empty paths
       - Return `Ok(())` for valid paths

    4. **`gateway_forward()` function** (pub(crate)):
       ```rust
       pub(crate) async fn gateway_forward(
           state: &AppState,
           method: reqwest::Method,
           path: &str,
           body: Option<Value>,
       ) -> Result<Value, AppError>
       ```
       - Check `openclaw_api_url(state)` -- if None, return `AppError::BadRequest("OpenClaw API not configured. Set OPENCLAW_API_URL in Settings > Connections.".into())` (use 503 via a new ServiceUnavailable variant, or use Internal with a known message -- see below)
       - Actually: since AppError has no 503 variant and adding one is scope creep, return `Err(AppError::Internal(anyhow::anyhow!("OpenClaw API not configured")))`. BUT the Internal variant's `into_response()` always returns "Something went wrong" to the client (line 43 of error.rs). So instead, use `AppError::BadRequest("OpenClaw API not configured. Set OPENCLAW_API_URL in Settings > Connections.".into())` which preserves the message for the frontend. This is semantically acceptable: the user hasn't configured it, which is a client-side issue.
       - Call `validate_gateway_path(path)?`
       - Build URL: `format!("{base_url}{path}")` where base_url is from `openclaw_api_url()`
       - Build reqwest request with `state.http.request(method, &url)`
       - Add `Authorization: Bearer {key}` header using `openclaw_api_key(state)` (only if key is non-empty)
       - Add `Content-Type: application/json` header
       - If body is Some, attach `.json(&body)`
       - Send with 30s timeout via `.timeout(Duration::from_secs(30))`
       - On send error: log with `tracing::error!` (redacted), return `AppError::Internal(anyhow!("OpenClaw gateway unreachable"))`
       - On response:
         - If status is success (2xx): parse JSON and return `Ok(value)`
         - If status is 404: return `AppError::NotFound("Resource not found on OpenClaw gateway".into())`
         - If status is 4xx: read body text, sanitize with `sanitize_error_body()`, return `AppError::BadRequest(sanitized_message)`
         - If status is 5xx: read body text, sanitize, log full error with `tracing::error!`, return `AppError::Internal(anyhow!("OpenClaw gateway error"))` (which becomes "Something went wrong" to the client -- safe)

    5. **Health-check route** `GET /api/openclaw/health`:

       - If `openclaw_api_url` is None: return `Ok(Json(json!({"ok": false, "status": "not_configured"})))`

    6. **Router**: `pub fn router() -> Router<AppState>` with `.route("/openclaw/health", get(openclaw_health))`
