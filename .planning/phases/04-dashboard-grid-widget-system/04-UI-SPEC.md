## Grid System Contract

### Column Calculation (Claude's Discretion)

Grid auto-calculates column count based on the dashboard container width (the main content area inside LayoutShell, after sidebar).

| Container Width | Columns | Column Width | Rationale |
|----------------|---------|-------------|-----------|
| < 600px | 1 | 100% | Narrow/mobile: single column stack |
| 600px -- 899px | 2 | ~50% each | Compact: sidebar collapsed, limited space |
| 900px -- 1399px | 3 | ~33% each | Default: standard desktop, matches existing 3-col grid |
| >= 1400px | 4 | ~25% each | Wide: ultrawide monitors, large windows |

**Implementation:** react-grid-layout's `useResponsiveLayout` hook with breakpoints:
```typescript
const breakpoints = { xl: 1400, lg: 900, md: 600, sm: 0 }
const cols = { xl: 4, lg: 3, md: 2, sm: 1 }
```

**Row height:** 80px per row unit. Combined with 16px margin, this produces:
- S widget (1x1): 80px tall, ~25% wide (at 4 cols)
- M widget (2x1): 80px tall, ~50% wide
- L widget (2x2): 176px tall (80*2 + 16 gap), ~50% wide
- XL widget (4x2): 176px tall, ~100% wide

### Widget Preset Sizes (Claude's Discretion)

| Preset | Grid Units (w x h) | Approximate Pixels (at 4-col, 1400px) | Use Case |
|--------|--------------------|-----------------------------------------|----------|
| S | 1 x 2 | ~330px x 176px | Status indicators (AgentStatusCard, HeartbeatCard) |
| M | 2 x 2 | ~676px x 176px | Standard cards (MissionsCard, MemoryCard) |
| L | 2 x 3 | ~676px x 272px | Data-rich cards (AgentsCard, NetworkCard) |
| XL | 4 x 3 | 100% x 272px | Full-width cards (IdeaBriefingCard with detail panel) |

**Free resize:** After placing with a preset, users can resize to any grid-aligned dimension. Minimum: 1x1. No maximum.

### Existing Card Size Mapping

| Card | Default Preset | Default Grid (w x h) | Rationale |
|------|---------------|----------------------|-----------|
| AgentStatusCard | S | 1 x 2 | Compact status display |
| HeartbeatCard | S | 1 x 2 | Compact status display |
| AgentsCard | M | 2 x 3 | List of agents needs vertical space |

### Non-Overlapping + Push Behavior

Widgets push each other down when dragged into occupied space (react-grid-layout default `compactType: 'vertical'`). Vertical compaction keeps widgets stacked without gaps.

**Pinned/floating widgets (z-order):** Widgets can be pinned to float above the grid. Pinned widgets:
- Render at `z-index: var(--z-dropdown)` (200) -- above grid but below modals
- Have a fixed position relative to the viewport (not the grid scroll)
- Show a pin icon in their title bar
- Do NOT push other widgets -- they overlay
- Cast an extra-high shadow to visually separate from the grid

### Scrollable Dashboard

Dashboard pages scroll vertically. The grid can be infinitely long. No horizontal scroll. The scrollable container is the main content area within LayoutShell.

## Widget Registry Architecture

### WidgetPlugin Interface

```typescript
interface WidgetPlugin {
  id: string                       // Unique widget ID: "agent-status", "heartbeat", etc.
  name: string                     // Display name: "Agent Status", "Heartbeat Monitor"
  description: string              // Short description for picker
  icon: React.ElementType          // Phosphor icon component
  category: 'monitoring' | 'productivity' | 'ai' | 'media' | 'custom'
  tier: 'builtin' | 'user' | 'ai' // Trust level
  component: React.LazyExoticComponent<React.ComponentType<WidgetProps>>
  configSchema?: WidgetConfigSchema  // Per-widget settings definition
  defaultSize: { w: number; h: number }
  minSize?: { w: number; h: number }
  maxSize?: { w: number; h: number }
  metadata?: {
    author?: string
    version?: string
    requiresService?: string       // "bluebubbles" | "openclaw" | etc.
  }
}

interface WidgetProps {
  widgetId: string                 // Instance ID (unique per placement)
  config: Record<string, unknown>  // Per-instance config values
  isEditMode: boolean              // Whether dashboard is in edit mode
  size: { w: number; h: number }   // Current grid size
}

interface WidgetConfigSchema {
  fields: Array<{
    key: string
    label: string
    type: 'text' | 'number' | 'toggle' | 'select' | 'slider'
    default: unknown
    options?: Array<{ label: string; value: unknown }>
    min?: number
    max?: number
  }>
}
```

### Three-Tier Trust Model

| Tier | Source | Trust Level | Rendering |
|------|--------|-------------|-----------|
| Built-in | Shipped with the app | Full trust | Direct React component in main context |
| User | Downloaded/installed (future Phase) | Validated | Direct React component after validation |
| AI | Bjorn-generated (Phase 7) | Sandboxed | iframe with `sandbox="allow-scripts"`, no same-origin |

Phase 4 only implements `builtin` tier. The registry interface is designed to support all three tiers for forward compatibility.

## Interaction Contracts

### Edit Mode

**Enter edit mode via (CONTEXT.md -- three ways):**
1. **Header button:** Pencil icon button in the dashboard header area. Toggles to a "Done" checkmark when active.
2. **Long-press on any widget:** 500ms press-and-hold on a widget triggers edit mode (iOS pattern). Visual feedback: widget grows slightly (`scale(1.01)`) during the hold, then snaps into wobble mode.
3. **Keyboard shortcut:** `Ctrl+E` (registered in keybindings.ts).

**Exit edit mode via:**
1. Click "Done" button (replaces pencil icon when in edit mode)
2. Press `Ctrl+E` again (toggle)
3. Press `Escape`

**Visual changes on entering edit mode (CONTEXT.md decisions):**


- **Grid lines:** Subtle dashed column/row guidelines. Color: `var(--border)` with `0.4` opacity. Pattern: `1px dashed`. Visible only in edit mode. Fade in over 200ms.
- **Wobble animation (iOS-style, toggleable per CONTEXT.md):** All widgets wobble simultaneously. Duration configurable: "off" in Settings disables the animation class.
- **Resize handles:** Small circles (8px diameter) at each corner and center of each edge. Color: `var(--accent)` at 60% opacity. Visible only in edit mode.
- **Remove X button:** Top-right corner of each widget, 24px diameter circle, background `var(--bg-elevated)`, icon color `var(--text-muted)`. On hover: background `var(--red-500)`, icon color `var(--text-on-color)`. Positioned 50% outside the widget border (offset: -8px, -8px).
- **Config gear icon:** Top-left corner of each widget (in edit mode) or on hover (in normal mode). 20px, `var(--text-muted)`, `var(--accent)` on hover.
- **Floating "+" button:** Bottom-right corner of dashboard, fixed position. 48px circle, `var(--accent)` background, `var(--text-on-accent)` Plus icon (24px). Shadow: Medium. On hover: `var(--accent-bright)` background, `scale(1.05)`.

### Wobble Animation (Claude's Discretion)

```css
@keyframes widget-wobble {
  0%   { transform: rotate(-0.5deg); }
  25%  { transform: rotate(0.5deg); }
  50%  { transform: rotate(-0.3deg); }
  75%  { transform: rotate(0.3deg); }
  100% { transform: rotate(-0.5deg); }
}

.widget-wobble {
  animation: widget-wobble 0.4s ease-in-out infinite;
}

/* Stagger wobble timing so widgets don't move in sync */
.widget-wobble:nth-child(odd)  { animation-delay: 0ms; }
.widget-wobble:nth-child(even) { animation-delay: 100ms; }
.widget-wobble:nth-child(3n)   { animation-delay: 200ms; }
```

- **Duration:** 0.4s per cycle, infinite loop
- **Rotation range:** +/- 0.5 degrees (subtle, iOS-like)
- **Stagger:** Each widget gets a slightly different start offset to avoid synchronized motion
- **Disable:** `data-wobble="off"` attribute on `<html>` disables the animation (Settings toggle, persisted in localStorage `wobble-enabled`)
- **Reduced motion:** Wobble disabled entirely per `prefers-reduced-motion: reduce` and `data-animation="reduced"` / `data-animation="none"` from Phase 3

### Long-Press to Enter Edit Mode

1. User presses and holds on any widget for 500ms
2. During hold (0-500ms): widget subtly scales up `transform: scale(1.01)` with `var(--ease-spring)` easing
3. At 500ms: haptic feedback (if available via Tauri), edit mode activates, all widgets begin wobbling
4. Releasing before 500ms: scale returns to 1.0, no edit mode change
5. Moving the pointer during hold: cancels the long-press (prevents accidental activation while scrolling)

### Add Widget -- Three Ways (CONTEXT.md)

#### 1. Floating "+" Button Opens Widget Picker Panel

The "+" button at bottom-right opens a slide-in panel from the right edge.

#### 2. Drag from Widget Picker onto Grid

When the widget picker panel is open, widgets can be dragged from the picker directly onto the grid. A ghost preview follows the cursor showing the widget's size preset. Dropping onto the grid places the widget.

#### 3. Click Empty Grid Space

### Undo Stack (Ctrl+Z)

Every edit mode action (move, resize, remove, add, config change) pushes to an undo stack. `Ctrl+Z` pops the most recent action and reverses it.

**Undo toast:** When an action is undone, show a toast: "Undone: {action description}" (e.g., "Undone: removed Heartbeat Monitor"). Auto-dismiss after 3 seconds.

**Stack depth:** 30 actions maximum. Stack clears when exiting edit mode (to prevent confusion about stale undo history).

### Persistence & Sync (CONTEXT.md decisions)

**Storage architecture:**

```typescript
interface DashboardLayoutState {
  pages: Array<{
    id: string               // UUID
    name: string             // "Home", "Work", "Monitoring"
    sortOrder: number
    layouts: {
      // Per-breakpoint layouts (PITFALLS.md #4 prevention)
      xl: LayoutItem[]       // 4-column layout
      lg: LayoutItem[]       // 3-column layout
      md: LayoutItem[]       // 2-column layout
      sm: LayoutItem[]       // 1-column layout
    }
    widgetConfigs: Record<string, Record<string, unknown>>  // Widget instance configs
  }>
  recycleBin: Array<{
    widgetId: string
    pluginId: string
    removedAt: string        // ISO timestamp
    previousPosition: LayoutItem
    previousPageId: string
  }>
  settings: {
    wobbleEnabled: boolean
    dotIndicatorsEnabled: boolean
  }
  lastModified: string       // ISO timestamp for sync conflict resolution
}

interface LayoutItem {
  i: string                  // Widget instance ID
  x: number                  // Grid x position
  y: number                  // Grid y position
  w: number                  // Width in grid units
  h: number                  // Height in grid units
  static?: boolean           // Pinned/immovable
  minW?: number
  minH?: number
}
```

**Sync strategy (from PITFALLS.md #4 and CONTEXT.md):**
1. **Local-first:** Layout saved to localStorage on every change (debounced 500ms)
2. **Supabase sync:** Saved to `user_preferences` via `preferences-sync.ts` (debounced 2s)
3. **Per-breakpoint:** Each breakpoint has its own layout. Changing layout at one breakpoint does not affect others.
4. **Last-write-wins with timestamp:** `lastModified` field resolves conflicts. Most recent edit wins.
5. **Auto-reflow option:** Users can toggle "Auto-reflow to smaller breakpoints" which re-calculates compact layouts from the widest layout instead of maintaining independent layouts.
