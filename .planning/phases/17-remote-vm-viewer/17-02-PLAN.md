**Create `frontend/src/pages/remote/VncViewer.tsx`:**

A component that renders the VNC canvas and the floating toolbar. Props:

```typescript
interface VncViewerProps {
  quality?: number
  compression?: number
}
```

Implementation:
1. Create a `containerRef = useRef<HTMLDivElement>(null)` for the noVNC canvas target
2. Track `quality` and `compression` in state (initialized from props, default 6 and 2)
3. Track `scale` mode in state: `'fit' | 'native'` (default `'fit'`)
4. Call `useVnc(containerRef, { quality, compression })` to get `{ connected, error, disconnect, reconnect, sendClipboard }`
5. Render structure:
   - Outer div: `position: relative; width: 100%; height: 100%; overflow: hidden; background: var(--bg-primary)`
   - VNC container div: `ref={containerRef}`, `style: { width: '100%', height: '100%' }` -- noVNC creates a canvas inside this
   - `<VncToolbar>` overlaid on top (position absolute)
   - If `error`, show a centered error overlay with the error message and a "Reconnect" button
   - If not connected and no error, show a centered "Connecting..." overlay with an amber spinner
6. Clipboard paste handler: Read from `navigator.clipboard.readText()` and call `sendClipboard(text)`
7. Fullscreen handler: `containerRef.current?.requestFullscreen()` (with fallback for webkit)
8. Export as named export: `export function VncViewer(props: VncViewerProps)`

**Create `frontend/src/pages/remote/VncToolbar.tsx`:**

Floating toolbar overlaid at the top center of the VNC viewer. Auto-hides after 3 seconds of mouse inactivity, reappears when mouse moves to the top 60px of the viewer area.

Props:
```typescript
interface VncToolbarProps {
  connected: boolean
  onDisconnect: () => void
  onReconnect: () => void
  onPasteClipboard: () => void
  onToggleScale: () => void
  onFullscreen: () => void
  scale: 'fit' | 'native'
  quality: number
  onQualityChange: (q: number) => void
}
```

Implementation:
1. State: `visible` (boolean, default true)
2. Ref: `hideTimerRef` for the auto-hide timeout
3. useEffect: Start a 3-second timer on mount and whenever `visible` becomes true. On timeout, set `visible = false`. Clear timer on unmount.
4. Mouse move handler on the parent VncViewer: When mouse Y is < 60px from top of the viewer, set `visible = true` and restart the hide timer
5. Toolbar renders as a horizontal bar:
   - `position: absolute; top: 12px; left: 50%; transform: translateX(-50%); z-index: 10`
   - Background: `var(--bg-secondary)` with `backdrop-filter: blur(12px)`, rounded corners, subtle shadow
   - Transition: `opacity` and `transform` for show/hide animation
   - When hidden: `opacity: 0; transform: translateX(-50%) translateY(-20px); pointer-events: none`
6. Toolbar items (all `<button>` elements with `aria-label`):
   - **Status dot**: 8px circle -- green (`var(--green-400)`) when connected, red (`var(--red-500)`) when disconnected, amber (`var(--amber)`) when connecting (derived from connected prop + no error = connecting)
   - **Disconnect/Reconnect button**: Shows "Disconnect" when connected, "Reconnect" when disconnected
   - **Paste clipboard button**: Clipboard icon, calls `onPasteClipboard`
   - **Scale toggle**: Shows "Fit" or "1:1" text, calls `onToggleScale`
   - **Quality dropdown**: `<select>` with options Low (2), Medium (5), High (8), calls `onQualityChange`
   - **Fullscreen button**: Expand icon, calls `onFullscreen`
7. Use CSS variables for all colors (no hardcoded values)
8. All buttons are `<button>` elements (not divs) with `aria-label` for accessibility

Do NOT use `<div onClick>` for any interactive element -- use `<button>` per CLAUDE.md accessibility rules.
Do NOT hardcode colors -- use CSS variables from globals.css.
