Create `src-tauri/src/routes/vnc.rs` with:

1. **CAS connection guard** (max 2 concurrent VNC sessions -- each is heavyweight):
   - `static VNC_CONNECTIONS: AtomicUsize` + `const MAX_VNC_CONNECTIONS: usize = 2`
   - `VncConnectionGuard` struct with `try_new()` CAS loop and `Drop` impl
   - Same pattern as `PtyConnectionGuard` in terminal.rs

2. **WebSocket upgrade handler** (`GET /api/vnc/ws`):
   - Handler signature: `async fn ws_upgrade(State(state): State<AppState>, RequireAuth(_session): RequireAuth, ws: WebSocketUpgrade) -> Response`
   - Return bare `Response` (NOT `Result<Response, AppError>`) -- Axum route gotcha documented in CLAUDE.md
   - Try to acquire VncConnectionGuard. If full, return `StatusCode::TOO_MANY_REQUESTS` with JSON `{"error": "too many VNC sessions (max 2)"}`
   - Call `ws.max_message_size(256 * 1024).on_upgrade(move |socket| handle_vnc_ws(socket, state, guard))`

3. **VNC WebSocket handler** (`handle_vnc_ws`):
   - Read `VNC_HOST` from `state.secret("VNC_HOST")`. If missing/empty, log error and return
   - Use `tokio::time::timeout(Duration::from_secs(5), TcpStream::connect(&vnc_host))` -- fail fast if VNC unreachable
   - On TCP connect failure, send a Text frame `{"error": "VNC server unreachable"}` to the WebSocket before returning
   - Split TCP stream: `let (tcp_reader, tcp_writer) = tcp_stream.into_split()`
   - Split WebSocket: `let (mut ws_sender, mut ws_receiver) = socket.split()`
   - **TCP-to-WS direction** (tokio::spawn): Read from `BufReader::new(tcp_reader)` into a `vec![0u8; 16384]` buffer, send as `Message::Binary(buf[..n].to_vec())`. Break on read 0 or send error.
   - **WS-to-TCP direction** (tokio::spawn): Receive from ws_receiver. On `Message::Binary(data)`, write to `BufWriter::new(tcp_writer)` and flush. Break on Close or error. Ignore non-binary messages.
   - Use `tokio::select!` to wait for either direction to finish, then log "vnc: session ended"
   - The `_guard: VncConnectionGuard` is moved into the closure -- it drops when the handler returns, decrementing the counter

4. **Status endpoint** (`GET /api/vnc/status`):
   - `async fn vnc_status(RequireAuth(_session): RequireAuth) -> Json<Value>`
   - Returns `{"active": N, "max": 2, "available": M}`

5. **Credentials endpoint** (`GET /api/vnc/credentials`):
   - `async fn vnc_credentials(State(state): State<AppState>, RequireAuth(_session): RequireAuth) -> Json<Value>`
   - Returns `{"password": state.secret_or_default("VNC_PASSWORD")}`
   - Password only crosses localhost (same security model as MC_API_KEY)

6. **Router function**:
   - `pub fn router() -> Router<AppState>` with three routes: `/api/vnc/ws`, `/api/vnc/status`, `/api/vnc/credentials`

7. **Route registration in mod.rs**:
   - Add `pub mod vnc;` to the module declarations (alphabetically near `vault`)
   - Add `.merge(vnc::router())` to the router chain

Required imports: `axum::{extract::{ws::{Message, WebSocket}, State, WebSocketUpgrade}, response::{IntoResponse, Response}, routing::get, Json, Router}`, `futures::{SinkExt, StreamExt}`, `serde_json::{json, Value}`, `std::sync::atomic::{AtomicUsize, Ordering}`, `std::time::Duration`, `tokio::io::{AsyncReadExt, AsyncWriteExt, BufReader, BufWriter}`, `tokio::net::TcpStream`, `tracing::{error, info}`, `crate::server::{AppState, RequireAuth}`

Do NOT use `Result<Response, AppError>` for the WS upgrade handler -- use bare `Response`.
Do NOT parse or inspect VNC protocol frames -- the proxy is a dumb binary pipe.
Do NOT use `std::io::Read` -- use `tokio::io::AsyncReadExt` (TcpStream is natively async, unlike portable-pty).
