  <action>
Create `scripts/check-bundle-size.sh` that validates JS bundle sizes from a pre-existing Vite build output directory.

The script MUST:

1. Accept an optional argument `--dist-dir` defaulting to `frontend/dist`
2. Find all `.js` files in `dist/assets/` and measure their uncompressed size in bytes using `stat` or `wc -c`
3. Define two thresholds as variables at the top of the script:
   - `MAX_CHUNK_KB=400` (single chunk limit, uncompressed)
   - `MAX_TOTAL_KB=5120` (total bundle limit, 5MB = 5120KB, uncompressed)
4. For each JS file, calculate size in KB. If any chunk exceeds `MAX_CHUNK_KB`, print a FAIL line naming the file and its size, and set exit code 1
5. Sum all JS file sizes. If total exceeds `MAX_TOTAL_KB`, print a FAIL line with the total and set exit code 1
6. On success, print a summary table of all chunks with sizes and a PASS message
7. Use colored output (RED for fail, GREEN for pass) matching the style in `pre-commit.sh`
8. Exit 0 on success, 1 on any violation
9. Do NOT run `vite build` itself — the script only validates an already-built dist directory. This allows both CI and pre-commit to call it after their own build step without double-building.

Script header:
```bash
#!/usr/bin/env bash
# Bundle size budget check — fails if any JS chunk > 400KB or total > 5MB (uncompressed)
set -euo pipefail
```

Use `stat --format='%s'` on Linux (fallback to `stat -f '%z'` on macOS for portability).
Use `awk` for KB conversion: `size_kb=$(( size_bytes / 1024 ))`.

The output format for each chunk should be:
```
  [chunk-name.js]  123 KB  (limit: 400 KB)  OK
  [big-chunk.js]   450 KB  (limit: 400 KB)  FAIL
```

And for total:
```
  Total JS: 2,345 KB  (limit: 5,120 KB)  OK
```

```bash
# 5b. Bundle size budget check
step_start=$(date +%s%N 2>/dev/null || python3 -c 'import time; print(int(time.time()*1e9))')

budget_output=$(bash "$ROOT/scripts/check-bundle-size.sh" --dist-dir "$FRONTEND/dist" 2>&1) || budget_exit=$?
budget_exit=${budget_exit:-0}
ms=$(elapsed_ms "$step_start")

if [ "$budget_exit" -eq 0 ]; then
  pass "$ms" "Bundle size budget (all chunks < 400KB, total < 5MB)"
else
  fail "$ms" "Bundle size budget"
  echo "$budget_output" | grep -E "FAIL|Total" | head -10 | while read -r line; do printf "      %s\n" "$line"; done
fi
```

This reuses the existing `pass`/`fail`/`section` helpers and `FAILED` variable from pre-commit.sh.
