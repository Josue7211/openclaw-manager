<objective>
Rewrite the OpenClaw gateway WebSocket handshake to use protocol v3 with proper identity, fix error response parsing, and surface protocol info in the UI.

Purpose: The current gateway connection sends a minimal connect message with only `auth.type: "password"` and `auth.password`. The real OpenClaw gateway protocol v3 requires `minProtocol/maxProtocol: 3`, `role: "operator"`, scopes, client metadata (id, version, platform, mode), and `auth.token`. Error responses also use `{ error: { message: "..." } }` format, not flat strings. This phase makes the handshake correct so all subsequent gateway RPC work (phases 77-90) operates on a properly authenticated connection.

Output: Working protocol v3 handshake in gateway_ws.rs, protocol version exposed through /api/gateway/status, and "Connected (protocol v3)" visible in Settings > Status.
</objective>
