  <action>
Create `frontend/src/__tests__/route-audit.test.tsx` with these design decisions:

**Strategy: Mock AuthGuard + LayoutShell, render each page in isolation.**

The test must NOT try to render the full app tree (LayoutShell has too many side effects -- Sidebar, theme polling, Tauri IPC, etc.). Instead:

1. **Mock AuthGuard** to pass children through immediately (no auth check):
   ```
   vi.mock('@/components/AuthGuard', () => ({ default: ({ children }) => children }))
   ```

2. **Mock LayoutShell** to render only `<Outlet />` (strips Sidebar, title bar, etc.):
   ```
   vi.mock('@/components/LayoutShell', () => {
     const { Outlet } = await vi.importActual('react-router-dom')
     return { default: () => <Outlet /> }
   })
   ```

3. **Mock common data-fetching modules** that pages call on mount:
   - `@/lib/api` -> return `{ api: { get: vi.fn().mockResolvedValue({}), post: vi.fn(), ... } }`
   - `@/lib/supabase/client` -> return `{ supabase: { from: () => ({ select: () => ({ data: [], error: null }) }) } }`
   - `@/lib/demo-data` -> return `{ isDemoMode: () => false, getDemoData: () => ({}) }`
   - `@/lib/preferences-sync` -> return `{ initPreferencesSync: vi.fn() }`

4. **Mock Tauri APIs** that pages may import:
   - Set `window.__TAURI_INTERNALS__ = undefined` to force browser-mode code paths.

5. **Define the full route list** as a const array matching main.tsx, with metadata:
   ```typescript
   const ROUTES: Array<{ path: string; type: 'page' | 'redirect'; target?: string }> = [
     { path: '/login', type: 'page' },
     { path: '/', type: 'page' },
     { path: '/personal', type: 'redirect', target: '/' },
     { path: '/dashboard', type: 'page' },
     // ... all 31 routes
   ]
   ```

6. **Sync guard test**: Read `main.tsx` source, extract all `path="..."` values via regex, and assert the test's ROUTES array covers them all. This prevents route drift (new route added to main.tsx but not to the test).

7. **For each page-type route**, use `it.each` or a loop with `describe`:
   - Render inside `<MemoryRouter initialEntries={[path]}>` with the same `<Routes>` structure from main.tsx (but with mocked AuthGuard/LayoutShell)
   - Use `waitFor` with 5000ms timeout for lazy components to resolve
   - Assert: `screen.queryByText('Something went wrong')` is null (no ErrorBoundary)
   - Assert: `screen.queryByText('This page crashed')` is null (no PageErrorBoundary)
   - Assert: `document.body.textContent` is not empty (not blank page)

8. **For redirect routes**, render and assert the final URL matches the target (check that the redirect target content renders, not the redirect itself).

9. **For catch-all route**, render `/nonexistent-route-xyz` and assert "404" or "Page not found" text appears.

**Important implementation notes:**
- Use `vi.mock` for all mocks (hoisted by vitest)
- Each route test should use `cleanup()` or rely on RTL auto-cleanup
- Import the route tree from main.tsx? NO -- reconstruct it in the test file using the same lazy imports. This avoids importing main.tsx which has massive side effects (theme init, Tauri IPC, ReactDOM.createRoot). The sync guard test reads main.tsx as a string to verify coverage.
- Wrap each page in `<Suspense fallback={<div data-testid="loading">Loading</div>}>` and wait for the fallback to disappear
- Some pages may throw errors because they call hooks that need providers (QueryClient). Wrap in `<QueryClientProvider>`.
- The test uses `@testing-library/react` which is already a dependency.
