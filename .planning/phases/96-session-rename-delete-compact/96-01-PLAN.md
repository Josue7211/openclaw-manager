Add three handler functions and register them in the router. All additions go in gateway.rs.

**1. Add imports** at the top of gateway.rs (merge with existing `use axum::...` line):
```rust
use axum::extract::Path;
use axum::routing::{patch, post, delete};
use serde::Deserialize;
```

**2. Add PatchSessionBody struct** (above the handler functions, after the `sanitize_error_body` function):
```rust
#[derive(Debug, Deserialize)]
struct PatchSessionBody {
    label: Option<String>,
}
```

**3. Add `patch_session` handler** (before the Router section):
- Signature: `async fn patch_session(State(state): State<AppState>, RequireAuth(_session): RequireAuth, Path(key): Path<String>, Json(body): Json<PatchSessionBody>) -> Result<Json<Value>, AppError>`
- Validate key: `if key.is_empty() || key.len() > 100` return `AppError::BadRequest("invalid session key")`
- Extract label: `body.label.as_deref().unwrap_or("").trim().to_string()`
- Call `gateway_forward(&state, Method::PATCH, &format!("/sessions/{key}"), Some(json!({ "label": label })))`
- Map errors: 4xx pass through as BadRequest, others become `"Gateway error: failed to rename session"`
- Return `Json(json!({ "ok": true, "data": payload }))`

**4. Add `delete_session` handler:**
- Signature: `async fn delete_session(State(state): State<AppState>, RequireAuth(_session): RequireAuth, Path(key): Path<String>) -> Result<Json<Value>, AppError>`
- NO Json body extractor -- key is in the path
- Same key validation (1-100 length)
- Call `gateway_forward(&state, Method::DELETE, &format!("/sessions/{key}"), None)`
- Map errors: `"Gateway error: failed to delete session"`
- Return `Json(json!({ "ok": true, "data": payload }))`

**5. Add `compact_session` handler:**
- Signature: `async fn compact_session(State(state): State<AppState>, RequireAuth(_session): RequireAuth, Path(key): Path<String>) -> Result<Json<Value>, AppError>`
- Same key validation (1-100 length)
- Call `gateway_forward(&state, Method::POST, &format!("/sessions/{key}/compact"), None)`
- Map errors: `"Gateway error: failed to compact session"`
- Return `Json(json!({ "ok": true, "data": payload }))`

**6. Register routes** in the `router()` function -- add two new `.route()` calls:
```rust
.route("/gateway/sessions/:key", patch(patch_session).delete(delete_session))
.route("/gateway/sessions/:key/compact", post(compact_session))
```

These do NOT conflict with the existing `.route("/gateway/sessions", get(gateway_sessions))` because `/gateway/sessions` (no param) and `/gateway/sessions/:key` (with param) are distinct Axum paths.

**7. Add unit tests** in the existing `#[cfg(test)] mod tests` block:
- `validate_path_accepts_session_key_paths`: assert `validate_gateway_path("/sessions/sess-123")` and `validate_gateway_path("/sessions/sess-123/compact")` are both Ok
- `validate_path_rejects_empty_after_sessions`: assert `validate_gateway_path("/sessions/")` is Ok (the handler validates key length separately -- the path validator only checks for `..`, `?`, `#`, `\0`, `\n`, `\r`)
