The hook `useChatSend(sessionKey: string | null, onSessionCreated?: (newKey: string) => void)` must:

1. Store `isSending`, `streamingMessage`, `sendError` in useState
2. Keep `streamingIdRef` (useRef) to track the active stream ID
3. Keep `currentSessionKeyRef` (useRef) synced with the sessionKey param

4. Call `useGatewaySSE` UNCONDITIONALLY (not inside if/else). Pass `undefined` if demo mode, otherwise pass `{ events: ['chat'], onEvent }`.

5. In the `onEvent` callback:
   - Guard: if `streamingIdRef.current` is null, return (no active stream)
   - Parse payload defensively: `const p = payload as Record<string, unknown>`
   - Token detection: `const token = (p.token ?? p.delta ?? p.content) as string | undefined`
   - If token exists and is a string, append to streamingMessage.content via `setStreamingMessage(prev => prev ? { ...prev, content: prev.content + token } : null)`
   - End-of-stream detection: `if (p.done === true || p.finish_reason === 'stop' || p.type === 'done')`
     - Set `streamingMessage.isStreaming = false`
     - Clear `streamingIdRef.current = null`
     - Invalidate `queryKeys.sessionHistory(currentSessionKeyRef.current!)` if session key exists
   - Add `console.debug('[chat-sse]', payload)` for first-run observation of actual payload shape

6. `sendMessage(text)` callback:
   - Guard: return early if demo, text.trim() empty, or isSending
   - `setSendError(null)`, `setIsSending(true)`
   - Generate `idempotencyKey = crypto.randomUUID()`
   - Generate `tempId = 'streaming-' + Date.now()`
   - Set `streamingIdRef.current = tempId`
   - Set streamingMessage: `{ id: tempId, role: 'assistant', content: '', isStreaming: true, timestamp: new Date().toISOString() }`
   - try: call `api.post<{ ok: boolean; data?: { sessionKey?: string } }>('/api/gateway/chat/send', { session_key: currentSessionKeyRef.current || null, message: text.trim(), deliver: true, idempotency_key: idempotencyKey })`
   - On success: check `result?.data?.sessionKey`. If it exists AND `currentSessionKeyRef.current` was null (new session), update `currentSessionKeyRef.current`, invalidate `queryKeys.gatewaySessions`, call `onSessionCreated?.(newKey)`
   - catch: clear streamingIdRef, clear streamingMessage, set sendError
   - finally: `setIsSending(false)`
