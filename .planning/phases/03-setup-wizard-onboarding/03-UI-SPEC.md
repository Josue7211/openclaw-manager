## Interaction Contracts

**Trigger:** App detects no `setup-complete` key in localStorage on mount.
**Pattern:** Full-screen overlay at `var(--z-wizard)` (10000), covers entire viewport including title bar.
**Background:** `var(--bg-base)` with ambient glow gradient.
**Dismiss:** Cannot be dismissed -- must complete, skip to demo, or skip wizard entirely.
**Focus trap:** Yes, via `useFocusTrap` on the wizard container.
**Keyboard:** Tab navigates form fields and buttons; Enter submits current step; Escape does nothing (no dismissal).

### Step Dots Progress Indicator

**Position:** Top center, 32px from top of viewport.
**Anatomy:** 10 dots (one per step: Welcome, Tailscale, Supabase, OpenClaw, Mac Services, Server Services, Modules, Theme, Summary, Done).
**Dot size:** 10px diameter.
**Dot gap:** 12px between dots.

| Dot State | Visual |
|-----------|--------|
| Completed | Filled circle, `var(--accent)` background |
| Completed + success | Filled circle, `var(--green)` background (service steps that passed test) |
| Completed + failed/skipped | Filled circle, `var(--text-muted)` background with 50% opacity |

```css
}
/* Duration: 2s, infinite, ease-in-out */
```

### Step Transitions (Claude's Discretion)

**Chosen approach: Morphing card transition.**

Each step is rendered inside a card-like container. When navigating forward, the current step's content fades out while scaling down slightly (0.96), then the container morphs its height to fit the new content, and the new step fades in while scaling up from 1.04. Going backward reverses the direction (scale up/down flipped). This creates a "breathing" effect that feels organic and modern, distinct from generic slides.

**Forward transition:**
1. Current content: `opacity 1 -> 0`, `scale(1) -> scale(0.96)`, `translateY(0) -> translateY(8px)` -- 200ms, `var(--ease-out)`
2. Container height: animates to new step height -- 250ms, `var(--ease-spring)`
3. New content: `opacity 0 -> 1`, `scale(1.04) -> scale(1)`, `translateY(-8px) -> translateY(0)` -- 250ms, `var(--ease-spring)`, 100ms delay after container resize starts

**Backward transition:**
1. Current content: `opacity 1 -> 0`, `scale(1) -> scale(1.04)`, `translateY(0) -> translateY(-8px)` -- 200ms, `var(--ease-out)`
2. Container height: animates to new step height -- 250ms, `var(--ease-spring)`
3. New content: `opacity 0 -> 1`, `scale(0.96) -> scale(1)`, `translateY(8px) -> translateY(0)` -- 250ms, `var(--ease-spring)`, 100ms delay

**Total perceived transition time:** ~350ms forward, ~350ms backward.

**Reduced motion:** Steps swap instantly (`opacity: 0 -> 1`, no transform, no height animation).

**Logo reveal animation (Claude's discretion):**

The logo starts as a single glowing accent-colored point at center. It expands outward as a radial wipe, revealing the full logo shape. Simultaneously, the logo fades from `var(--accent)` monochrome to full color (or stays accent-tinted if logo is monochrome). A subtle glow halo expands behind the logo and fades.

```css
@keyframes logo-reveal {
  0% {
    clip-path: circle(0% at 50% 50%);
    filter: brightness(2) saturate(0.5);
    opacity: 0;
  }
  30% {
    opacity: 1;
  }
  100% {
    clip-path: circle(100% at 50% 50%);
    filter: brightness(1) saturate(1);
    opacity: 1;
  }
}

@keyframes logo-glow {
  0% { box-shadow: 0 0 0 0 rgba(var(--accent-rgb), 0.3); }
  50% { box-shadow: 0 0 40px 20px rgba(var(--accent-rgb), 0.15); }
  100% { box-shadow: 0 0 60px 30px rgba(var(--accent-rgb), 0); }
}
```

- **Logo reveal duration:** 800ms, `var(--ease-spring)`
- **Glow halo duration:** 1200ms, `var(--ease-out)`, starts 300ms after logo begins
- **Heading fade-in:** 400ms delay after logo starts, `opacity 0 -> 1 + translateY(12px) -> translateY(0)`, 400ms, `var(--ease-spring)`
- **Subheading fade-in:** 200ms after heading starts
- **Buttons stagger:** 100ms between each, starting 200ms after subheading

**Reduced motion:** Logo appears instantly at full opacity, no glow. Text appears instantly. No stagger.

### Celebration Animation (Claude's Discretion)

**Chosen approach: Confetti burst + gentle fade transition.**

When the user clicks the final "Next" to reach the summary screen:
1. The step transition plays as normal
2. Once the summary content fades in, a confetti burst fires from the center-top of the screen
3. Confetti uses `canvas-confetti` library: 100 particles, spread 70 degrees, origin `{ x: 0.5, y: 0.3 }`, colors derived from the active theme (`--accent`, `--green`, `--secondary`, `var(--text-primary)` at 30% alpha)
4. Confetti falls with gravity over 3 seconds, then canvas cleans up
5. The "You're all set!" heading has a subtle scale entrance: `scale(0.9) -> scale(1)` over 500ms, `var(--ease-spring)`

**Reduced motion:** No confetti. Heading appears instantly without scale animation.

### Dashboard Transition (Summary to Live App)

When clicking "Launch Dashboard":
1. Wizard container fades out: `opacity 1 -> 0`, 300ms, `var(--ease-out)`
2. Simultaneously, wizard scales down slightly: `scale(1) -> scale(0.95)`, 300ms
3. `setup-complete` is written to localStorage
4. Router navigates to `/` (dashboard)
5. Normal app shell renders with standard page entrance animation

**Reduced motion:** Instant swap, no animation.

### Animation Intensity Setting

**Storage key:** `animation-intensity` in localStorage.
**Values:** `'full'` | `'reduced'` | `'none'`
**Default:** `'full'` unless `prefers-reduced-motion: reduce` is detected, in which case default is `'reduced'`.

| Level | Behavior |
|-------|----------|
| Full | All animations play at full duration and effect |
| Reduced | Transitions shortened to 100ms, no bouncing/spring easing, no confetti, no logo reveal glow, step transitions are simple fades |
| None | No animations at all. All transitions instant. `* { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }` |

**Implementation:**
- `useAnimationIntensity()` hook returns current level
- Components check the level and conditionally apply animation classes
- A CSS class on `<html>`: `data-animation="full"`, `data-animation="reduced"`, `data-animation="none"`
- In Settings > Display (not in the wizard itself -- CONTEXT says the toggle is in Settings, not wizard)

## Wizard State Architecture

### State Shape

```typescript
interface WizardState {
  currentStep: number               // 0-9 (Welcome=0, Done=9)
  completedSteps: Set<number>       // Steps that have been successfully completed
  stepStatus: Record<number, 'idle' | 'testing' | 'success' | 'error' | 'skipped'>

  // Service credentials (temporary, saved to keychain on completion)
  tailscaleIp: string
  supabaseUrl: string
  supabaseAnonKey: string
  openclawUrl: string
  openclawApiKey: string
  blueBubblesUrl: string
  blueBubblesPassword: string
  macBridgeUrl: string
  macBridgeApiKey: string
  couchdbUrl: string
  couchdbUsername: string
  couchdbPassword: string

  // Test results
  testResults: Record<string, {
    status: 'idle' | 'testing' | 'success' | 'error'
    latencyMs?: number
    error?: string
  }>

  // Module selection
  enabledModules: string[]
  activeBundle: 'essentials' | 'full' | 'minimal' | null

  // Theme selection
  selectedThemeId: string
  selectedMode: 'dark' | 'light' | 'system'
}
```

### Persistence

- **Storage key:** `wizard-state`
- **Pattern:** `useSyncExternalStore` (matches theme-store, sidebar-settings patterns)
- **Serialization:** JSON.stringify/parse, excluding `testResults` (tests re-run on resume)
- **Credentials:** Temporarily held in wizard state during setup, saved to OS keychain only on wizard completion (not on each step)
- **Cleanup:** `wizard-state` key deleted from localStorage when wizard completes
