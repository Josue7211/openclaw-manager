## Typography

Inherited from Phase 1 -- no structural changes to the scale. Phase 2 makes all typography theme-controlled via `--font-body`, `--font-heading`, `--font-mono`, `--font-ui` CSS variables, plus a base font size multiplier.

| Role | CSS Variable | Size | Weight | Line Height | Usage |
|------|-------------|------|--------|-------------|-------|
| Caption | `--text-sm` | 13px | 400 | 1.4 | Timestamps, swatch labels, schedule time labels, share code preview |
| Body | `--text-base` | 15px | 400 | 1.5 | Theme card names, font picker items, import/export descriptions, setting labels |
| Heading | `--text-xl` | 20px | 600 | 1.2 | Picker section headings ("Dark Themes", "Light Themes"), Settings panel headings |
| Display | `--text-2xl` | 24px | 600 | 1.2 | Theme picker modal title |

### Base Font Size Multiplier

A global scale slider (80% to 120%) multiplies all type sizes. Implementation: set `font-size` on `<html>` element as a percentage, and use `rem` units for all type sizes. Default: 100%.

- Minimum: 80% (equivalent to 12.8px base)
- Maximum: 120% (equivalent to 19.2px base)
- Step: 5% increments
- Stored with theme, synced via preferences-sync

### Font Application Rules

- **`--font-body`**: All paragraph text, list items, descriptions, nav items, button text
- **`--font-heading`**: Page titles, section headings, card titles, modal titles
- **`--font-mono`**: Code blocks, editor content, metadata, technical values
- **`--font-ui`**: Sidebar labels, button text, form labels, tab labels, badge text, toast text. Falls back to `--font-body` if not set.

## Color

### Two-Tier Variable Architecture (RESEARCH.md Pitfall #9 prevention)

Phase 2 introduces a two-tier color architecture to make presets maintainable:

**Tier 1 -- Primitive variables** (fixed per base theme, never changed by accent selection):
- Surface: `--bg-base`, `--bg-panel`, `--bg-card`, `--bg-card-hover`, `--bg-elevated`, `--bg-card-solid`
- Text: `--text-primary`, `--text-secondary`, `--text-muted`
- Border: `--border`, `--border-hover`, `--border-strong`, `--border-subtle`
- Glass: `--glass-bg`, `--glass-border`, `--glass-blur`
- Hover: `--hover-bg`, `--hover-bg-bright`, `--active-bg`
- Overlays: `--overlay-light`, `--overlay`, `--overlay-heavy`
- Popover/Modal: `--bg-popover`, `--bg-modal`

**Tier 2 -- Semantic variables** (accent-dependent, user can override independently):
- `--accent`, `--accent-dim`, `--accent-bright`
- `--accent-secondary`, `--accent-secondary-dim`, `--accent-secondary-bright`
- `--glow-top-rgb`
- `--logo-color`
- `--border-accent`
- All `--*-a{N}` alpha tint variables (derived from accent color programmatically)

## Theme Architecture

### ThemeDefinition Type

```typescript
interface ThemeDefinition {
  id: string                    // "dracula", "nord", "custom-{uuid}"
  name: string                  // "Dracula", "My Custom Theme"
  category: 'dark' | 'light' | 'high-contrast' | 'colorful'
  builtIn: boolean              // true for shipped presets, false for user/imported
  artwork?: string              // Base64 data URL or asset path for picker card
  colors: {
    // Tier 1 -- surfaces, text, borders (~20 properties)
    'bg-base': string
    'bg-panel': string
    'bg-card': string
    'bg-card-hover': string
    'bg-elevated': string
    'bg-card-solid': string
    'text-primary': string
    'text-secondary': string
    'text-muted': string
    'border': string
    'border-hover': string
    'border-strong': string
    'border-subtle': string
    'glass-bg': string
    'glass-border': string
    'hover-bg': string
    'hover-bg-bright': string
    'active-bg': string
    'bg-popover': string
    'bg-modal': string
    // Tier 2 -- accent defaults (overridable by user)
    'accent': string
    'accent-dim': string
    'accent-bright': string
    'glow-top-rgb': string
    // Status colors
    'green': string
    'red': string
    'red-500': string
    'warning': string
    [key: string]: string       // Additional overrides
  }
  fonts?: {
    body?: string
    heading?: string
    mono?: string
    ui?: string
  }
  fontScale?: number            // 0.8 to 1.2 (base font size multiplier)
}

interface UserThemeOverrides {
  themeId: string               // Which base theme this overrides
  accent?: string               // Custom accent hex
  secondary?: string            // Custom secondary hex
  glow?: string                 // Custom glow hex
  logo?: string                 // Custom logo color hex
  fonts?: {
    body?: string
    heading?: string
    mono?: string
    ui?: string
  }
  fontScale?: number
  pinned?: boolean              // Favorite/pin in picker
}

interface ThemeState {
  mode: 'dark' | 'light' | 'system'
  activeThemeId: string         // Currently active base theme
  overrides: Record<string, UserThemeOverrides>  // Per-theme user overrides keyed by theme ID
  customThemes: ThemeDefinition[]                // User-created/imported themes
  schedule?: ThemeSchedule
  pageOverrides?: Record<string, string>         // Route path -> theme ID
  categoryOverrides?: Record<string, string>     // Category ID -> theme ID
  globalFontOverride?: boolean                   // If true, font stays when switching themes
}
```

### Theme Application Flow

1. Determine active theme: `mode === 'system'` ? check `prefers-color-scheme` to pick light/dark default : use `activeThemeId`
2. Load `ThemeDefinition.colors` -- set each as CSS custom property on `document.documentElement`
3. Apply `UserThemeOverrides` on top (accent, glow, secondary, logo, fonts)
4. Derive alpha tint variables programmatically from accent color (using `color-mix()` where supported, fallback to JS `hexToRgba()`)
5. Apply fonts to `--font-body`, `--font-heading`, `--font-mono`, `--font-ui`
6. Apply font scale to `<html>` font-size
7. Set `data-theme="light"` or `data-theme="dark"` on `<html>` for any remaining CSS selectors
8. Check per-page/per-category overrides and apply as scoped CSS variables if applicable

### State Management Pattern

Use `useSyncExternalStore` (same pattern as `keybindings.ts`, `sidebar-settings.ts`):
- Store: `lib/theme-store.ts`
- Persistence: localStorage (immediate) + Supabase via `preferences-sync.ts` (debounced 2s)
- SYNCED_KEYS additions: `'theme-state'` (replaces separate `'theme'` and `'accent-color'` keys)
- Migration: existing `'theme'` and `'accent-color'` keys migrate into the new `ThemeState` structure on first load

### Theme Switching Animation (Ripple)

**Effect:** A circular clip-path animation that expands from the click point, revealing the new theme underneath.

**Implementation approach:**
1. On theme click, capture `event.clientX`, `event.clientY`
2. Create a clone of the current page state as a `<div>` overlay (or use `document.startViewTransition()` if available)
3. Apply new theme CSS variables to the real DOM
4. Animate the overlay clip-path from `circle(0px at ${x}px ${y}px)` to `circle(${maxRadius}px at ${x}px ${y}px)`
5. Remove overlay when animation completes

**Timing:**
- Duration: 400ms
- Easing: `var(--ease-spring)` -- `cubic-bezier(0.22, 1, 0.36, 1)`
- System-follow OS switch: ripple from center of screen (`50vw, 50vh`)

**CSS keyframe:**
```css
@keyframes theme-ripple {
  from { clip-path: circle(0px at var(--ripple-x) var(--ripple-y)); }
  to   { clip-path: circle(var(--ripple-radius) at var(--ripple-x) var(--ripple-y)); }
}
```

**Reduced motion:** Falls back to instant swap (no animation) per existing `prefers-reduced-motion: reduce` rule.

### Import/Export Panel

**Location:** Settings > Display > Import/Export section, also accessible from "+" button in theme picker

#### Import Methods

| Method | Interaction |
|--------|------------|
| **File picker** | Click "Import Theme" button, OS file dialog opens, accepts `.json` files only |
| **Paste JSON/URL** | Text area input, paste raw JSON or a URL pointing to a theme JSON file |
| **Drag and drop** | Drop a `.json` file onto the theme picker modal or import area |
| **Share code** | Text input for base64-encoded compressed theme string |

**Validation on import:**
- Parse JSON, verify `ThemeDefinition` structure
- Reject if contains `url()`, `@import`, `expression()`, or external resource references (PITFALL #13)
- Only accept recognized CSS property names from whitelist
- Show clear error message on validation failure: "This theme file contains unsupported properties and can't be imported. Only standard theme files are accepted."
- Successful import: theme appears in "Custom" section of picker, auto-selected

#### Export

- **JSON file:** Download as `{theme-name}.json` containing full `ThemeDefinition` + all user overrides
- **Share code:** Generate compressed base64 string via lz-string, copy to clipboard. Display in a selectable text field.
- **Artwork toggle:** Checkbox "Include artwork image" (default: checked). Unchecked produces smaller export.

#### Share Code Format

```
ocm-theme:v1:{lz-string compressed base64 of JSON}
```

Prefix `ocm-theme:v1:` enables version detection and prevents accidental import of unrelated data.

**Location:** Settings > Display > Custom CSS section

**Two tabs:**

### Theme Scheduling

**Location:** Settings > Display > Schedule section

#### Sunset/Sunrise Auto-Switch
- Toggle: "Auto switch at sunrise/sunset"
- Light theme dropdown: select which theme to use during day
- Dark theme dropdown: select which theme to use at night
- Sunrise/sunset calculated from system timezone offset (no location API)
- Approximate times displayed: "Sunrise ~6:30 AM, Sunset ~7:45 PM" (updates seasonally)

#### Manual Time Ranges
- Toggle: "Use custom schedule"
- Table of time ranges:
  ```
  [6:00 AM] to [6:00 PM]  [Solarized Light  v]
  [6:00 PM] to [6:00 AM]  [Nord             v]
  ```
- "Add time range" button
- Validation: ranges must cover 24 hours, no gaps, no overlaps

**Schedule state:**
```typescript
interface ThemeSchedule {
  type: 'sunrise-sunset' | 'manual' | 'none'
  sunriseSunset?: {
    dayThemeId: string
    nightThemeId: string
  }
  manual?: Array<{
    startHour: number  // 0-23
    startMinute: number
    endHour: number
    endMinute: number

  }>
```

## Security Contract (Theme Import/Export)

### Validation Whitelist

Theme imports ONLY accept CSS custom property overrides matching these patterns:
- `bg-*`, `text-*`, `border-*`, `accent*`, `glow-*`, `glass-*`, `hover-*`, `active-*`
- `green*`, `red*`, `warning`, `amber*`, `yellow*`, `gold*`, `blue*`, `purple*`, `cyan*`, `pink*`, `orange*`
- `shadow-*`, `overlay*`
- `font-body`, `font-heading`, `font-mono`, `font-ui`

### Rejection Patterns

Imported theme JSON is rejected if any value contains:
- `url(` -- external resource loading
- `@import` -- external stylesheet loading
- `expression(` -- IE expression evaluation
- `javascript:` -- script protocol
- `<script` or `</script` -- HTML injection
- `var(--` referencing variables NOT in the whitelist
- Values longer than 200 characters (prevents data exfiltration via long strings)

### Custom CSS Safety

Custom CSS injection (built-in editor and external file) is NOT sanitized -- it is an explicit power-user feature. Safety is provided by:
1. Existing CSP (`img-src 'self' data: http://127.0.0.1:3000`) blocks external resource loading in CSS
2. Dedicated `<style>` element that can be instantly removed
3. Warning banner in the editor UI
4. "Clear Custom CSS" one-click reset
