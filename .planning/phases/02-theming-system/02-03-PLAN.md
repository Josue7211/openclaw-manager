1. Create `frontend/src/lib/theme-validation.ts`:

   **Property name whitelist** (regex patterns matching allowed CSS variable names without -- prefix):
   ```typescript
   const ALLOWED_PROPERTY_PATTERNS = [
     /^bg-/, /^text-/, /^border-/, /^accent/, /^glow-/, /^glass-/,
     /^hover-/, /^active-/, /^green/, /^red/, /^warning/, /^amber/,
     /^yellow/, /^gold/, /^blue/, /^purple/, /^cyan/, /^pink/, /^orange/,
     /^shadow-/, /^overlay/, /^font-(body|heading|mono|ui)$/,
   ]
   ```

   **Dangerous value patterns** (reject if any color value matches):
   ```typescript
   const DANGEROUS_PATTERNS = [
     /url\s*\(/i, /@import/i, /expression\s*\(/i, /javascript:/i,
     /<script/i, /<\/script/i,
   ]
   ```

   **validateThemeImport(data: unknown): { valid: boolean; error?: string }**
   - Check `data` is non-null object
   - Check required fields: `id` (string), `name` (string), `colors` (object)
   - Check `category` is one of 'dark'|'light'|'high-contrast'|'colorful' if present
   - For each entry in `colors`:
     - Check key matches at least one ALLOWED_PROPERTY_PATTERN
     - Check value is a string
     - Check value length <= 200
     - Check value does NOT match any DANGEROUS_PATTERN
   - If fonts provided, validate it's an object with optional string values for body/heading/mono/ui only
   - If fontScale provided, validate it's a number between 0.5 and 1.5
   - Return `{ valid: true }` if all pass, or `{ valid: false, error: '...' }` with specific message

   **exportTheme(theme: ThemeDefinition, overrides?: UserThemeOverrides, includeArtwork?: boolean): string**
   - Create payload object: `{ theme: { ...theme, builtIn: false }, overrides }`
   - If `includeArtwork === false`, delete `payload.theme.artwork`
   - Return `JSON.stringify(payload, null, 2)`

   **encodeShareCode(theme: ThemeDefinition, overrides?: UserThemeOverrides): string**
   - Import `LZString` from 'lz-string'
   - Create payload: `{ theme: { ...theme, builtIn: false, artwork: undefined }, overrides }`
   - `const json = JSON.stringify(payload)`
   - `return 'ocm-theme:v1:' + LZString.compressToBase64(json)`

   **decodeShareCode(code: string): { theme: ThemeDefinition; overrides?: UserThemeOverrides } | null**
   - Check prefix: `if (!code.startsWith('ocm-theme:v1:')) return null`
   - Extract compressed portion: `code.slice('ocm-theme:v1:'.length)`
   - `const json = LZString.decompressFromBase64(compressed)`
   - If null or empty, return null
   - Try JSON.parse, return null on failure
   - Return parsed `{ theme, overrides }` (caller should still validateThemeImport on theme)

   **downloadThemeJson(theme: ThemeDefinition, overrides?: UserThemeOverrides, includeArtwork?: boolean): void**
   - Call `exportTheme(theme, overrides, includeArtwork)`
   - Create Blob with type 'application/json'
   - Create object URL, create hidden `<a>` element, set `download` to `${theme.name.toLowerCase().replace(/\s+/g, '-')}.json`
   - Click, cleanup

   **parseImportInput(input: string): { theme: ThemeDefinition; overrides?: UserThemeOverrides } | null**
   - Trim input
   - If starts with `'ocm-theme:v1:'`, call `decodeShareCode(input)`
   - Else, try `JSON.parse(input)`:
     - If parsed has `.theme` key, return `{ theme: parsed.theme, overrides: parsed.overrides }`
     - If parsed has `.id` and `.name` and `.colors`, return `{ theme: parsed }`
   - Return null on any failure

2. Create `frontend/src/lib/__tests__/theme-validation.test.ts` with all behavior tests above. Use real lz-string (no mocking) for share code round-trip tests.
