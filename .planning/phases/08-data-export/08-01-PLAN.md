Create `src-tauri/src/routes/export.rs` with three handler functions and a `pub fn router() -> Router<AppState>`:

**1. `GET /api/export/supabase` — export_supabase handler:**
- Extract `RequireAuth(session)` and `State(state)`
- Build a `SupabaseClient` via `SupabaseClient::from_state(&state)?`
- Define `EXPORT_TABLES` as a const array matching the user-data tables from SYNC_TABLES, but EXCLUDING `cache` (internal sync data, not user content) and `user_preferences` (settings, not meaningful export data). Include: `todos`, `missions`, `mission_events`, `agents`, `ideas`, `captures`, `habits`, `habit_entries`, `changelog_entries`, `decisions`, `knowledge_entries`, `daily_reviews`, `weekly_reviews`, `retrospectives`, `workflow_notes`
- Also include `pipeline_events` and `activity_log` (user data tables from the initial migration that are not in SYNC_TABLES)
- Loop over EXPORT_TABLES, call `supabase.select_as_user(table, "select=*", &session.access_token).await`
- Collect results into a `serde_json::Map` keyed by table name
- Wrap failures per-table as `{ "error": "message" }` rather than failing the entire export (best-effort)
- Add metadata: `{ "exported_at": chrono::Utc::now().to_rfc3339(), "user_id": session.user_id, "tables": {...} }`
- Return `Ok(Json(json!({ "ok": true, "data": export_obj })))` using success_json pattern
- Log with `tracing::info!(tables = EXPORT_TABLES.len(), "export: supabase data exported")`

**2. `GET /api/export/sqlite` — export_sqlite handler:**
- Extract `RequireAuth(_session)` and `State(state)`
- Resolve SQLite database path using `state.app.path().app_data_dir()` from Tauri path resolver, then append `db.sqlite`
- Read the file bytes using `tokio::fs::read(&db_path).await`
- Return raw bytes as response with headers:
  - `Content-Type: application/octet-stream`
  - `Content-Disposition: attachment; filename="mission-control-backup.sqlite"`
- Use `axum::response::Response` with `axum::body::Body::from(bytes)` to stream the file
- If file not found, return `AppError::NotFound("SQLite database not found")`
- Log with `tracing::info!(size_bytes = bytes.len(), "export: sqlite backup sent")`

**3. `GET /api/export/notes` — export_notes handler:**
- Extract `RequireAuth(_session)` and `State(state)`
- Replicate the CouchDB bulk-fetch pattern from vault.rs `list_notes` but simplified:
  - Call the vault `couch_config` helper. Since it's private in vault.rs, duplicate the helper as a local private function `vault_config(state: &AppState) -> Option<(String, String, String, String)>` that reads `COUCHDB_URL`, `COUCHDB_USER`, `COUCHDB_PASSWORD`, `COUCHDB_DATABASE` from state secrets
  - Call `state.http.get(format!("{url}/{db}/_all_docs?include_docs=true")).basic_auth(&user, Some(&pass)).send().await`
  - Filter docs: skip `_design/*`, `h:*`, `ps:*`, `ix:*`, `cc:*`, `.obsidian`, `!:`, `!_` prefixes (same as vault.rs)
  - Skip attachments (binary files) via the same extension check
  - Build chunk map from `h:*` docs for content reassembly
  - For each note doc, reassemble content from children + eden (same logic as vault.rs list_notes)
  - Return array of `{ "id": doc._id, "content": reassembled_markdown }` objects
- If CouchDB not configured, return `Ok(success_json(json!({ "notes": [], "message": "CouchDB not configured" })))`
- Log with `tracing::info!(notes = count, "export: notes exported as markdown")`
