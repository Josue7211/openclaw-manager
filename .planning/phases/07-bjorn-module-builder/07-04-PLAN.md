    2. Create `frontend/src/lib/bjorn-store.ts`:

       **Internal state:**
       - `_blobUrls` Map<string, string> -- tracks blob URLs per module id for cleanup
       - `_registered` Set<string> -- tracks which module ids are currently registered in widget registry

       **Helper functions:**
       - `wrapAsESModule(source: string): string` -- Wraps the generated component source as an ES module. Appends `\nexport default BjornWidget;` to the source. This is the contract: Bjorn-generated code must define a function named BjornWidget.
       - `exposePrimitivesAPI(): void` -- Sets `(window as any).__bjornAPI` to an object containing references to all 11 primitive components (StatCard, LineChart, BarChart, etc.) so blob URL modules can access them. Import each primitive component and assign to the global. Call this once at startup before loading modules.

       **Core registration:**
       - `registerBjornModule(module: BjornModule): void`
         1. If `_blobUrls.has(module.id)`, revoke the old blob URL first: `URL.revokeObjectURL(_blobUrls.get(module.id)!)`
         2. Create new Blob from `wrapAsESModule(module.source)` with type 'application/javascript'
         3. Create blob URL via `URL.createObjectURL(blob)`
         4. Store in `_blobUrls.set(module.id, url)`
         5. Call `registerWidget({ id: 'bjorn-' + module.id, name: module.name, description: module.description, icon: module.icon || 'Cube', category: 'custom', tier: 'ai', defaultSize: module.defaultSize, configSchema: module.configSchema, component: () => import(/* @vite-ignore */ url), metadata: { author: 'Bjorn', version: String(module.version) } })`
         6. Add to `_registered.add(module.id)`

       - `unregisterBjornModule(moduleId: string): void`
         1. Revoke blob URL if exists
         2. Remove from `_blobUrls` and `_registered`
         3. NOTE: Widget registry does not have an unregister function. The widget remains in the Map but won't appear in picker if we add a `deleted` flag or filter. For now, register a dummy "deleted" placeholder that renders nothing -- OR just leave it (the module won't be in any layout so it won't render). Leave it for simplicity.

       **API-backed CRUD:**
       - `loadBjornModules(): Promise<void>` -- `api.get<{ modules: BjornModule[] }>('/api/bjorn/modules')`, then for each module where `module.enabled`, call `registerBjornModule(module)`. Wrap in try/catch -- non-fatal on failure (log warning).
       - `saveBjornModule(data: { name, description, icon, source, configSchema, defaultSize }): Promise<BjornModule>` -- `api.post('/api/bjorn/modules', data)`, then `registerBjornModule(result.module)`, return module.
       - `updateBjornModule(id: string, data: { source, configSchema?, name?, description?, icon? }): Promise<BjornModule>` -- `api.put('/api/bjorn/modules/' + id, data)`, then `registerBjornModule(result.module)`, return module.
       - `deleteBjornModule(id: string): Promise<void>` -- `api.del('/api/bjorn/modules/' + id)`, then `unregisterBjornModule(id)`.
       - `toggleBjornModule(id: string, enabled: boolean): Promise<BjornModule>` -- `api.patch('/api/bjorn/modules/' + id + '/toggle', { enabled })`. If enabled, `registerBjornModule(result.module)`. If disabled, `unregisterBjornModule(id)`. Return module.
       - `rollbackBjornModule(id: string, version: number): Promise<BjornModule>` -- `api.post('/api/bjorn/modules/' + id + '/rollback', { version })`, then `registerBjornModule(result.module)`, return module.
       - `getBjornVersions(id: string): Promise<BjornModuleVersion[]>` -- `api.get('/api/bjorn/modules/' + id + '/versions')`, return versions array.
