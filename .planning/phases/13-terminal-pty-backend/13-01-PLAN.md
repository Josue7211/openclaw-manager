From src-tauri/src/routes/chat.rs (CAS guard pattern):
```rust
static WS_CONNECTIONS: AtomicUsize = AtomicUsize::new(0);
const MAX_WS_CONNECTIONS: usize = 5;

struct WsConnectionGuard;
impl WsConnectionGuard {
    fn try_new() -> Option<Self> {
        loop {
            let current = WS_CONNECTIONS.load(Ordering::Acquire);
            if current >= MAX_WS_CONNECTIONS { return None; }
            if WS_CONNECTIONS.compare_exchange(current, current + 1, Ordering::AcqRel, Ordering::Acquire).is_ok() {
                return Some(Self);
            }
        }
    }
}
impl Drop for WsConnectionGuard {
    fn drop(&mut self) { WS_CONNECTIONS.fetch_sub(1, Ordering::AcqRel); }
}
```

**Section E: PTY command builder with env sanitization**
```rust
fn build_pty_command() -> CommandBuilder {
    let shell = detect_shell();
    let mut cmd = CommandBuilder::new(&shell);

    // CRITICAL: Clear ALL env vars first -- Tauri process has 30+ secrets
    cmd.env_clear();

    // Whitelist only safe system variables
    for key in PTY_SAFE_ENV_VARS {
        if let Ok(val) = std::env::var(key) {
            cmd.env(key, val);
        }
    }

    // Force TERM for proper terminal emulation (matches xterm.js in Phase 14)
    cmd.env("TERM", "xterm-256color");

    cmd
}
```
Note: The `env_clear()` approach is a whitelist -- only PTY_SAFE_ENV_VARS are forwarded. The PTY_BLOCKED_PREFIXES list exists for the unit test to verify NO blocked prefix vars leak through, providing defense-in-depth documentation.

**Section H: Bidirectional WebSocket-PTY relay**

`async fn handle_terminal_ws(socket: WebSocket, _guard: PtyConnectionGuard)`:

1. Open PTY with `native_pty_system().openpty(PtySize { rows: 24, cols: 80, pixel_width: 0, pixel_height: 0 })`.
2. Build command via `build_pty_command()`.
3. Spawn shell on slave: `pair.slave.spawn_command(cmd)`.
4. **Immediately `drop(pair.slave)`** -- CRITICAL to avoid PTY EOF hang.
5. Get process group leader: `pair.master.process_group_leader()` returns `Option<i32>` on Unix.
6. Clone reader: `pair.master.try_clone_reader().unwrap()`.
7. Take writer: `pair.master.take_writer().unwrap()`.
8. Store master + child in an `Arc<Mutex<Option<PtyCleanup>>>` struct for Drop-based cleanup.
9. Split WebSocket: `let (ws_sender, ws_receiver) = socket.split();`.
10. Wrap `ws_sender` in `Arc<Mutex<_>>` so both read task and main can use it.

**Read task** (PTY stdout -> WebSocket binary):
```rust
let read_handle = tokio::spawn(async move {
    let mut buf = vec![0u8; 4096];
    loop {
        let mut reader_guard = reader_mutex.lock().await;
        let reader_ref = &mut *reader_guard;
        // spawn_blocking because portable-pty Read is sync
        let result = {
            let mut r = reader_ref.take().expect("reader consumed");
            let res = tokio::task::spawn_blocking(move || {
                let n = r.read(&mut buf);
                (r, buf, n)
            }).await;
            // ... handle result
        };
        // If read returns Ok(0) or Err, break
        // Send as Message::Binary (NOT Text -- PTY output may have non-UTF-8 bytes)
    }
});
```

IMPORTANT: The reader is `Box<dyn Read + Send>`. It cannot be shared across threads. Use a pattern where the blocking task takes ownership and returns it. The simplest approach:

- Move the reader into a dedicated `std::thread::spawn` (OS thread, not tokio), which reads in a loop and sends bytes through a `tokio::sync::mpsc::channel(32)`.
- The async task receives from the channel and forwards to WebSocket as `Message::Binary`.
- This avoids the ownership dance with `spawn_blocking`.

**Section I: PtyCleanup struct with process group kill on Drop**
```rust
struct PtyCleanup {
    child: Box<dyn portable_pty::Child + Send>,
    #[allow(dead_code)]
    master: Box<dyn MasterPty + Send>,
    pgid: Option<i32>,
}

impl Drop for PtyCleanup {
    fn drop(&mut self) {
        // Graceful kill via portable-pty
        let _ = self.child.kill();

        // Kill entire process group (catches vim, htop, subprocesses)
        #[cfg(unix)]
        if let Some(pgid) = self.pgid {
            if pgid > 0 {
                unsafe { libc::kill(-pgid, libc::SIGKILL); }
            }
        }

        // Reap zombie
        let _ = self.child.wait();
    }
}
```
