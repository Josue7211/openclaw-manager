<objective>
Build the theme interpolation engine that blends all Tier 1 CSS variables between a dark and light theme at any position t in [0, 1]. Tier 2 accent/status colors remain untouched. Text color auto-switches based on blended background OKLCH lightness. Every text/background pair is WCAG AA compliant (4.5:1 contrast ratio) at every blend position.

Purpose: This is the computational core of the theme blend slider (Phase 8). Without this engine, the slider has nothing to drive.
Output: Exported `interpolateThemes()` function in theme-engine.ts, wired into `applyTheme()`, with comprehensive unit tests.
</objective>

    ```typescript
    const TIER_1_KEYS: ReadonlyArray<string> = [
      'bg-base', 'bg-panel', 'bg-card', 'bg-card-hover', 'bg-elevated', 'bg-card-solid',
      'bg-popover', 'bg-modal', 'text-primary', 'text-secondary', 'text-muted',
    ```

    Implement `parseColor(val: string): [number, number, number, number]`:
    - If val starts with '#', parse as hex: r=parseInt(slice(1,3),16), g=parseInt(slice(3,5),16), b=parseInt(slice(5,7),16), a=1
    - If val starts with 'rgba(' or 'rgb(', use regex `/rgba?\(\s*(\d+)\s*,\s*(\d+)\s*,\s*(\d+)(?:\s*,\s*([\d.]+))?\s*\)/` to extract r,g,b,a (a defaults to 1)
    - Fallback: return [0, 0, 0, 1]

    Implement `formatColor(r: number, g: number, b: number, a: number): string`:
    - If a === 1: return hex string `#${r.toString(16).padStart(2,'0')}${g.toString(16).padStart(2,'0')}${b.toString(16).padStart(2,'0')}`
    - Otherwise: return `rgba(${r}, ${g}, ${b}, ${round a to 2 decimals})`

    Implement `contrastRatio(fg: string, bg: string): number`:
    - Parse fg and bg to get [r,g,b,a] via parseColor
    - For each channel, compute relative luminance per WCAG: linearize (c/255: if <= 0.04045 then c/12.92 else ((c+0.055)/1.055)^2.4), then L = 0.2126*R + 0.7152*G + 0.0722*B
    - Contrast ratio = (lighter + 0.05) / (darker + 0.05) where lighter = max(L1,L2), darker = min(L1,L2)
    - For rgba values, composite against white for light colors or black for dark colors before computing luminance (simplification: use the rgb channels directly, ignoring alpha for contrast -- alpha primarily affects visual blending with the page behind it, not the semantic text/bg contrast)

    Implement helper `bgLightness(hexOrRgba: string): number`:
    - Parse the color value. If hex, use `hexToOklch(hex)[0]` (the L value).
    - If rgba, extract r,g,b, convert to hex first, then hexToOklch.
    - Returns OKLCH lightness L (0-1).

    Implement `interpolateThemes(darkDef: ThemeDefinition, lightDef: ThemeDefinition, t: number): Record<string, string>`:
    - Clamp t to [0, 1]
    - For each key in TIER_1_KEYS:
      - Get darkVal = darkDef.colors[key], lightVal = lightDef.colors[key]
      - If either is undefined, skip (don't include in output)
      - Parse both with parseColor -> [r1,g1,b1,a1] and [r2,g2,b2,a2]
      - Interpolate: r = Math.round(r1 + (r2-r1)*t), same for g, b. For a: a = round((a1 + (a2-a1)*t) * 100) / 100
      - Format back with formatColor
      - Store in result Record
    - **Text color auto-switch (per user decision):**
      - Compute bgL = bgLightness(result['bg-base'])
      - If bgL > 0.6: text should be dark. Set result['text-primary'] to the DARKER of the two theme's text-primary values (by OKLCH lightness). If neither is dark enough, use '#1d1d1f'. Same logic for text-secondary and text-muted.
      - If bgL <= 0.6: text should be light. Set result['text-primary'] to the LIGHTER of the two theme's text-primary values. If neither is light enough, use '#e4e4ec'. Same for text-secondary and text-muted.
    - **WCAG AA enforcement (post-interpolation nudge):**
      - For each text key ('text-primary', 'text-secondary', 'text-muted'):
        - Compute contrast = contrastRatio(result[textKey], result['bg-base'])
        - If contrast < 4.5: nudge the text color toward black (if bgL > 0.6) or white (if bgL <= 0.6) by darkening/lightening until 4.5:1 is met. Use a simple loop: step lightness by 0.05 in OKLCH space (via hexToOklch/oklchToHex), recheck contrast, max 20 iterations.
      - This ensures readability at every blend position including the muddy mid-range.
    - Return the blended Record (only Tier 1 keys, never Tier 2).

    Export: `parseColor`, `formatColor`, `contrastRatio`, `interpolateThemes` (all exported for testing).

    ```typescript
    // --- Theme blend interpolation ---
    const bp = state.blendPosition
    let blendedVars: Record<string, string> | null = null
    if (bp != null && bp > 0 && bp < 1) {
      // Find the counterpart theme to blend toward
      const counterpartId = COUNTERPART_MAP[def.id]
      if (counterpartId) {
        const counterpart = getThemeById(counterpartId)
        if (counterpart) {
          // Dark theme is always first arg, light theme second
          const isDark = def.category !== 'light'
          const darkDef = isDark ? def : counterpart
          const lightDef = isDark ? counterpart : def
          blendedVars = interpolateThemes(darkDef, lightDef, bp)
        }
      }
    }
    ```

    ```typescript
    for (const [key, value] of Object.entries(def.colors)) {
    ```
