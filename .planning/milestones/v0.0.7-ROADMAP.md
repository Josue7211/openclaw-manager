## Phases

### v0.0.7 -- AgentShell, Agent Secrets, and MemD

**Branch plan** *(one branch per workstream; keep PRs small and reviewable)*
- `chore/repo-pr-ops` - branch split policy, PR template, branch protection, autonomous runbook
- `feature/memd-brain` - durable memory contract, storage, retrieval, compaction
- `feature/agentshell-contract` - AgentShell adapter contract, safety validation, approval bridge
- `feature/agent-secrets` - scoped secret lifecycle, masking, audit, rotation, UI
- `feature/agent-platform-integration` - end-to-end verification and production build hardening

**PR order:**
1. Repo / PR operations and branch split policy
2. MemD brain contract and storage
3. AgentShell adapter contract
4. Agent secrets lifecycle and UI
5. Cross-system verification and release hardening

**Group AJ: Repo & PR Operations** *(branch discipline, release queue, and autonomous execution)*
- [ ] **Phase 99: Branch Split & PR Workflow** - Split work into topic branches, codify the PR template/checklist, and lock in the merge order for open-source review

**Group AK: MemD Foundation** *(the AI brain; durable memory, context retrieval, and compaction)*
- MemD contract draft: [milestones/v0.0.7-MEMD-CONTRACT.md](milestones/v0.0.7-MEMD-CONTRACT.md)
- [ ] **Phase 100: MemD Contract Definition** - Define persistence, retrieval, retention, and sync boundaries before implementation starts
- [ ] **Phase 101: MemD Storage & APIs** - Implement the backend surface and frontend consumers for memory access
- [ ] **Phase 102: MemD UX & Compaction** - Add review, prune, and compaction flows with guard rails for stale context

**Group AL: AgentShell Contract** *(typed adapter, explicit approvals, no secret authority)*
- [ ] **Phase 103: Typed AgentShell Request Contract** - Validate request payloads, normalize response envelopes, and reject unknown fields before proxying
- [ ] **Phase 104: AgentShell Launch & Approval Bridge** - Wire the adapter to a safe launch/approval flow with explicit consent and clear user feedback
- [ ] **Phase 105: AgentShell Safety Hardening** - Enforce allowlists, redaction, timeout handling, and regression tests for the proxy layer

**Group AM: Agent Secrets** *(scoped credentials, auditability, and controlled rotation)*
- [ ] **Phase 106: Secret Scope Model** - Bind secrets to an agent/service scope instead of a global bucket so access is explicit
- [ ] **Phase 107: Secret Lifecycle & Audit** - Add rotate/revoke flows, masked logs, and auditable access records
- [ ] **Phase 108: Secret Management UI** - Expose create/update/delete and permission-aware summaries in Settings

**Group AN: Cross-System Verification** *(ship only after the three surfaces work together)*
- [ ] **Phase 109: End-to-End Integration** - Verify AgentShell, secrets, and MemD behave correctly in one production build
- [ ] **Phase 110: Branch & PR Readiness** - Split remaining follow-ups into clean branches, document release order, and run final verification
