## Phases

### v0.0.7 -- AgentShell, Agent Secrets, and MemD

**Branch plan** *(one branch per workstream; keep PRs small and reviewable)*
- `feature/agentshell-contract` - AgentShell adapter contract, safety validation, approval bridge
- `feature/agent-secrets` - scoped secret lifecycle, masking, audit, rotation, UI
- `feature/memd-foundation` - durable memory contract, storage, retrieval, compaction
- `feature/agent-platform-integration` - end-to-end verification and production build hardening

**Group AJ: AgentShell Contract** *(typed adapter, explicit approvals, no secret authority)*
- [ ] **Phase 99: Typed AgentShell Request Contract** - Validate request payloads, normalize response envelopes, and reject unknown fields before proxying
- [ ] **Phase 100: AgentShell Launch & Approval Bridge** - Wire the adapter to a safe launch/approval flow with explicit consent and clear user feedback
- [ ] **Phase 101: AgentShell Safety Hardening** - Enforce allowlists, redaction, timeout handling, and regression tests for the proxy layer

**Group AK: Agent Secrets** *(scoped credentials, auditability, and controlled rotation)*
- [ ] **Phase 102: Secret Scope Model** - Bind secrets to an agent/service scope instead of a global bucket so access is explicit
- [ ] **Phase 103: Secret Lifecycle & Audit** - Add rotate/revoke flows, masked logs, and auditable access records
- [ ] **Phase 104: Secret Management UI** - Expose create/update/delete and permission-aware summaries in Settings

**Group AL: MemD Foundation** *(working name; durable memory layer, naming still provisional)*
- [ ] **Phase 105: MemD Contract Definition** - Define persistence, retrieval, retention, and sync boundaries before implementation starts
- [ ] **Phase 106: MemD Storage & APIs** - Implement the backend surface and frontend consumers for memory access
- [ ] **Phase 107: MemD UX & Compaction** - Add review, prune, and compaction flows with guard rails for stale context

**Group AM: Cross-System Verification** *(ship only after the three surfaces work together)*
- [ ] **Phase 108: End-to-End Integration** - Verify AgentShell, secrets, and MemD behave correctly in one production build
- [ ] **Phase 109: Branch & PR Readiness** - Split remaining follow-ups into clean branches, document release order, and run final verification
