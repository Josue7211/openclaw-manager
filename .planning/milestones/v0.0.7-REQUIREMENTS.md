# Requirements: OpenClaw Manager

**Defined:** 2026-04-04
**Core Value:** MemD is the AI brain of the product - durable memory and retrieval should drive the assistant experience, with AgentShell and agent secrets acting as the safety and capability layers around it.

## v0.0.7 Requirements

Requirements for the repo / PR ops work plus the AgentShell / agent secrets / MemD milestone. Each maps to roadmap phases.

### Repo / PR Operations

- [ ] **OPS-01**: Branch-split policy, PR template, and review queue are documented and used for every workstream
- [ ] **OPS-02**: GSD autonomous execution has a clear phase order, handoff protocol, and branch ownership model

### MemD Core

- [ ] **MEMD-01**: MemD defines the canonical durable-memory contract for storage, retrieval, retention, and compaction
- [ ] **MEMD-02**: MemD can surface relevant memory context to assistant workflows without exposing raw private data to the webview
- [ ] **MEMD-03**: MemD supports safe pruning, compaction, and stale-context cleanup without breaking active sessions
- [ ] **MEMD-04**: MemD has clear persistence boundaries so the brain layer can be audited and versioned independently

### AgentShell

- [ ] **SHELL-01**: AgentShell validates typed requests and rejects unknown fields before proxying
- [ ] **SHELL-02**: AgentShell supports explicit launch and approval flows with clear user consent
- [ ] **SHELL-03**: AgentShell remains a thin adapter and never becomes a second secret authority
- [ ] **SHELL-04**: AgentShell behavior is covered by regression tests for allowlists, redaction, and timeout handling

### Agent Secrets

- [ ] **SECRETS-01**: Secrets are scoped by agent/service purpose instead of a single global bucket
- [ ] **SECRETS-02**: Secret creation, rotation, and revocation are auditable and permission-aware
- [ ] **SECRETS-03**: Secret values are masked in logs, UI, and error paths
- [ ] **SECRETS-04**: Settings exposes a safe management UI for secret summaries and lifecycle actions

### Cross-System Integration

- [ ] **INTEG-01**: MemD, AgentShell, and agent secrets work together in one production build
- [ ] **INTEG-02**: Cross-system flows preserve isolation between memory, secrets, and transport boundaries
- [ ] **INTEG-03**: Final verification confirms the architecture is branchable, reviewable, and ready for PR-based open-source management

## Out of Scope

| Feature | Reason |
|---------|--------|
| Open-ended marketplace features | The milestone is about core architecture, not ecosystem expansion |
| Replacing the existing auth model | Use the current login and secret boundaries |
| Making AgentShell a product runtime | It stays an adapter layer, not the brain |
