  <behavior>
    StatCard:
    - Renders title text from config.title (default: "Metric")
    - Renders formatted value from config.value (default: "0")
    - Renders unit suffix from config.unit if provided
    - Renders trend arrow: up (green/secondary), down (red), flat (muted) based on config.trend ("up"/"down"/"flat")
    - Renders SVG sparkline polyline from config.data number array (min 2 points)
    - Shows EmptyState with ChartLineUp icon when no config.value and no config.data
    - Sparkline color resolves from config.color via resolveColor()
    - Exports configSchema with fields: title (text), value (text), unit (text), trend (select: up/down/flat), color (select: accent/secondary/tertiary/red/amber)

    ProgressGauge:
    - Renders linear progress bar when config.variant is "bar" (default)
    - Renders circular SVG gauge when config.variant is "circular"
    - Value from config.value (number, default 0), max from config.max (number, default 100)
    - Renders percentage label: Math.round((value / max) * 100) + "%"
    - Bar variant uses div with width percentage + theme color
    - Circular variant uses SVG circle with stroke-dasharray/dashoffset
    - Color resolves from config.color via resolveColor()
    - Shows label from config.label if provided
    - Exports configSchema with fields: label (text), value (number), max (number), variant (select: bar/circular), color (select)
  </behavior>
  <action>
    **StatCard.tsx:**
    - React.memo wrapped, implements WidgetProps
    - Co-exports configSchema: WidgetConfigSchema with 5 fields: title, value, unit, trend, color
    - Extract config using configString/configArray from shared.tsx
    - Trend indicator: ArrowUp/ArrowDown/Minus icons from @phosphor-icons/react, colored with CSS vars (var(--secondary) for up, var(--red) for down, var(--text-muted) for flat)
    - SVG sparkline: inline svg with viewBox, preserveAspectRatio="none", polyline computed from data points. Use width:100% height:40px. Map data to normalized y-coordinates (0=bottom, height=top).
    - Layout: flex column, title at top (14px, text-muted), value large (32px, text-primary, font-weight 700), trend+unit row, sparkline at bottom
    - Default export for React.lazy compatibility

    **ProgressGauge.tsx:**
    - React.memo wrapped, implements WidgetProps
    - Co-exports configSchema: WidgetConfigSchema with 5 fields: label, value, max, variant, color
    - Bar variant: outer div with var(--bg-base) background, inner div with calculated width%, resolveColor(config.color) background, border-radius from CSS vars, height 12px
    - Circular variant: SVG with two circle elements -- background circle (stroke: var(--border)) and foreground circle (stroke: resolveColor, stroke-dasharray/dashoffset for percentage). Use transform="rotate(-90)" on foreground for 12-o-clock start. Size derived from widget height.
    - Percentage label centered (for circular) or right-aligned (for bar)
    - Clamp value between 0 and max to prevent overflow

    **register.ts:**
    - Add registerWidget calls for both prim-stat-card and prim-progress-gauge:
      - prim-stat-card: category 'primitives', defaultSize {w:3, h:2}, minSize {w:2, h:2}, icon 'ChartLineUp'
      - prim-progress-gauge: category 'primitives', defaultSize {w:3, h:2}, minSize {w:2, h:1}, icon 'Gauge'
    - Import configSchema from each and pass to the registration

    **Tests:** Render each primitive with @testing-library/react, assert:
    - StatCard: title text rendered, sparkline SVG present, trend icon present, empty state when no data
    - ProgressGauge: bar div present with correct width style, circular SVG present, percentage label
  </action>
