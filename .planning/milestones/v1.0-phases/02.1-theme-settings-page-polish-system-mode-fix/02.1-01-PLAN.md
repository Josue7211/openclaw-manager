<objective>
Fix System mode dark detection on Linux (Hyprland/Wayland).

Purpose: Tauri's `getCurrentWindow().theme()` reads `gtk-application-prefer-dark-theme` which is unset on Hyprland and many Wayland compositors. System mode always reports "light" even when the user has a dark GTK theme. This makes the System toggle useless on Linux.

Output: A Rust Tauri command `detect_system_dark_mode` that checks `gsettings` as a fallback, and frontend wiring that calls it when native detection returns 'light' on Linux.
</objective>

Add a new `#[tauri::command]` function `detect_system_dark_mode() -> bool` to `commands.rs`.

On Linux (`#[cfg(target_os = "linux")]`):
1. First check `gsettings get org.gnome.desktop.interface color-scheme` -- if stdout contains "prefer-dark", return true.
2. Fallback: check `gsettings get org.gnome.desktop.interface gtk-theme` -- if stdout (lowercased) contains "dark", return true.
3. If both fail (gsettings not installed, or neither returns dark), return false.
4. Wrap ALL `Command::new("gsettings")` calls in proper error handling -- if the binary doesn't exist or returns non-zero, continue to next check. Never panic.

On non-Linux (`#[cfg(not(target_os = "linux"))]`): return false (Tauri native detection works on macOS/Windows).

Register the command in `main.rs` by adding `commands::detect_system_dark_mode` to the `generate_handler![]` macro.
