  <action>
**theme-definitions.ts:**
Add 3 optional fields to `UserThemeOverrides`:
```typescript
glowOpacity?: number    // 0 to 0.25 (controls --glow-opacity)
borderRadius?: number   // 0 to 24 (controls --radius-md in px)
panelOpacity?: number   // 0.4 to 1.0 (controls glass panel alpha)
```

**theme-store.ts:**
Add 3 new mutation functions following the exact pattern of `setAccentOverride`:

1. `setGlowOpacity(opacity: number)` — clamp to [0, 0.25], store in override, call applyThemeFromState()
2. `setBorderRadius(radius: number)` — clamp to [0, 24], store in override, call applyThemeFromState()
3. `setPanelOpacity(opacity: number)` — clamp to [0.4, 1.0], store in override, call applyThemeFromState()

**theme-engine.ts:**
Add `applyAdvancedOverrides(overrides: UserThemeOverrides | undefined, isLight: boolean)` function, called at the end of the `apply()` closure in `applyTheme()`:

1. **Glow opacity:** Set `--glow-opacity` to `overrides?.glowOpacity ?? (isLight ? 0.06 : 0.10)`. This is what `body::before` reads.
2. **Border radius:** If `overrides?.borderRadius` is defined, set `--radius-sm` to `Math.max(0, val - 4)px`, `--radius-md` to `${val}px`, `--radius-lg` to `${val + 4}px`, `--radius-xl` to `${val + 8}px`. This scales proportionally.
3. **Panel opacity:** If `overrides?.panelOpacity` is defined, compute the glass-bg value as the current theme's glass-bg color but with the alpha channel replaced. Set `--glass-opacity` to the value (for future use), and recalculate `--glass-bg` by parsing the existing `rgba()` and replacing the alpha. Simpler approach: set a `--glass-opacity-override` variable, and use it if present.

Actually, simplest approach for panel opacity: set `--glass-bg` directly. Parse the current theme's `glass-bg` value from `def.colors['glass-bg']` (which is like `rgba(18, 18, 24, 0.6)`), extract the RGB part, and reconstruct with the new alpha. Set both `--glass-bg` and `--bg-panel` with the new alpha.
