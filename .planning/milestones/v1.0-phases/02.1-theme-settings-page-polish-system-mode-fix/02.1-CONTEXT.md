# Phase 02.1: Theme Settings Page Polish + System Mode Fix - Context

**Gathered:** 2026-03-19
**Status:** Ready for planning
**Source:** User testing feedback (USER-FEEDBACK.md)

<domain>
## Phase Boundary

Fix System mode dark detection on Linux, redesign Settings > Display page for visual organization, and expand color customization with a proper 3-tier color hierarchy (accent → secondary/functional → tertiary). Make the app maximally customizable.

</domain>

<decisions>
## Implementation Decisions

### System Mode Detection (LOCKED)
- Tauri's getCurrentWindow().theme() reads gtk-application-prefer-dark-theme flag, which is often unset on Hyprland
- Need Rust Tauri command that checks `gsettings get org.gnome.desktop.interface gtk-theme` for "dark" substring as fallback
- Frontend calls this command, caches result, feeds into resolveThemeDefinition()

### Color Hierarchy (LOCKED — from user)
- **Primary/Accent** = main theme accent color (already works, --accent)
- **Secondary** = functional/status color. Currently hardcoded as --green everywhere. Must become customizable:
  - "Add" buttons, "RUNNING" badges, todo checkboxes, progress bars, success states, active indicators, Proxmox/OPNsense status
  - Default value: green (#34d399) — but user can change to any color
  - Maps to: --secondary (replaces hardcoded --green usage across the app)
- **Tertiary** = what's currently --accent-blue/--accent-secondary. Chat bubbles, dashboard accents, less prominent
  - Default: derived from accent, user can override
  - Maps to: --tertiary (replaces --accent-blue)
- **Glow** = ambient top glow gradient (--glow-top-rgb) — keep as separate picker
- **Logo** = sidebar logo color (--logo-color) — keep as separate picker

### Settings Display Page Redesign (LOCKED — from user)
- Card-based sections with clear headers for visual hierarchy
- Group: Mode Selector | Theme Presets | Colors (accent, secondary, tertiary, glow, logo) | Fonts | Advanced
- Color pickers should be compact — NOT repetitive rows of the same 7 swatches
- Font section needs live preview text that updates as font changes
- More customizability: border radius, opacity, spacing sliders

### Claude's Discretion
- Exact layout of the color picker section (compact grid vs inline pickers)
- How to handle border-radius/opacity/spacing sliders (which CSS vars to expose)
- Whether to add per-page color overrides to this page or keep in sidebar context menu
- Implementation details of the Rust gsettings command

</decisions>
