1. Create `frontend/src/lib/theme-engine.ts`:

   **resolveThemeDefinition(state: ThemeState): ThemeDefinition**
   - Look up `state.activeThemeId` in BUILT_IN_THEMES, then in `state.customThemes`
   - If `state.mode === 'system'`:
     - Check `window.matchMedia('(prefers-color-scheme: light)').matches`
     - If OS is light and resolved theme is dark-category: return `getThemeById('default-light')` (or first light built-in)
     - If OS is dark and resolved theme is light-category: return `getThemeById('default-dark')` (or first dark built-in)
   - Fallback: return `getThemeById('default-dark')!` if nothing matches

   **deriveAlphaTints(hex: string): void**
   - Parse hex to RGB: `const r = parseInt(hex.slice(1,3), 16)` etc.
   - Set on `document.documentElement.style`:
     - `--accent-a10: rgba(${r}, ${g}, ${b}, 0.1)` through all accent alpha tints
     - `--purple-a08` through `--purple-a90` (same RGB -- these are accent-derived)
     - `--border-accent: rgba(${r}, ${g}, ${b}, 0.25)`
   - Full list of accent-derived tints to regenerate:
     ```
     accent-a10: 0.1, accent-a12: 0.12, accent-a15: 0.15, accent-a30: 0.3, accent-a40: 0.4
     purple-a08: 0.08, purple-a10: 0.1, purple-a12: 0.12, purple-a15: 0.15, purple-a20: 0.2,
     purple-a30: 0.3, purple-a40: 0.4, purple-a55: 0.55, purple-a75: 0.75, purple-a90: 0.9
     border-accent: 0.25
     ```

   **applyFonts(fonts: { body?: string; heading?: string; mono?: string; ui?: string } | undefined, globalOverride?: boolean): void**
   - If fonts provided, set `--font-body`, `--font-heading`, `--font-mono`, `--font-ui` on documentElement
   - If `globalOverride` is true and no fonts provided, skip (keep current fonts)
   - Append system font fallbacks to each: e.g., `'${font}', -apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui, sans-serif` for body/heading/ui; monospace stack for mono

   **applyFontScale(scale: number | undefined): void**
   - Default scale is 1.0 if undefined
   - Clamp to [0.8, 1.2]
   - Multiply each --text-* variable by scale and set on documentElement:
     - `--text-2xs: ${Math.round(9 * scale)}px`
     - `--text-xs: ${Math.round(11 * scale)}px`
     - `--text-sm: ${Math.round(13 * scale * 10) / 10}px`
     - `--text-base: ${Math.round(15 * scale * 10) / 10}px`
     - `--text-lg: ${Math.round(17 * scale)}px`
     - `--text-xl: ${Math.round(20 * scale)}px`
     - `--text-2xl: ${Math.round(24 * scale)}px`
     - `--text-3xl: ${Math.round(28 * scale)}px`
   - NOTE: Do NOT change html font-size (Pitfall #5). Only multiply --text-* variables.

   **performRippleTransition(applyFn: () => void, x: number, y: number): void**
   - Use `document.startViewTransition()` exactly as in RESEARCH.md Pattern 3
   - Import `flushSync` from 'react-dom'
   - Calculate `maxRadius = Math.hypot(Math.max(x, right), Math.max(y, bottom))`
   - Animate `::view-transition-new(root)` pseudo-element with clip-path from `circle(0px at ${x}px ${y}px)` to `circle(${maxRadius}px at ${x}px ${y}px)`
   - Duration: 400ms, easing: `cubic-bezier(0.22, 1, 0.36, 1)`
   - Wrap in try/catch -- if any error, just call applyFn() directly

   **applyTheme(state: ThemeState, clickEvent?: { clientX: number; clientY: number }): void**
   The main orchestrator function:
   1. Call `resolveThemeDefinition(state)` to get the ThemeDefinition
   2. Get user overrides: `state.overrides[state.activeThemeId]`
   3. Create `apply` closure that:
      a. Iterates `def.colors` entries, sets each as `el.style.setProperty('--' + key, value)` on documentElement
      b. Applies user accent override via `applyAccentColor(overrides?.accent || def.colors.accent)`
      c. Applies glow override via `applyGlowColor(overrides?.glow || def.colors['glow-top-rgb'])`
      d. Applies secondary override via `applySecondaryColor(overrides?.secondary || def.colors['accent-secondary'] || DEFAULT_SECONDARY)`
      e. Applies logo override via `applyLogoColor(overrides?.logo || def.colors.accent)`
      f. Calls `deriveAlphaTints(overrides?.accent || def.colors.accent)`
      g. Sets `el.dataset.theme` = category includes 'light' ? 'light' : 'dark'
      h. Sets `el.dataset.themeId` = def.id
      i. Calls `applyFonts(overrides?.fonts || def.fonts, state.globalFontOverride)`
      j. Calls `applyFontScale(overrides?.fontScale || def.fontScale)`
   4. If `clickEvent` exists AND `document.startViewTransition` exists AND `!window.matchMedia('(prefers-reduced-motion: reduce)').matches`:
      - Call `performRippleTransition(apply, clickEvent.clientX, clickEvent.clientY)`
   5. Else: call `apply()` directly
