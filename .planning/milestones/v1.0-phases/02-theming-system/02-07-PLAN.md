1. Create `frontend/src/components/ThemeImportExport.tsx`:

   **Import section:**
   - "Import Theme" button: Opens OS file dialog (if Tauri, use `open()` from '@tauri-apps/plugin-dialog' with `{ filters: [{ name: 'Theme', extensions: ['json'] }] }`; if browser, use `<input type="file" accept=".json">`)

   - "Import" button next to paste area: Calls `parseImportInput(textareaValue)`, then `validateThemeImport(result.theme)`, then `addCustomTheme(result.theme)`. Show toast on success/error.
   - Drag and drop: On the modal/section, `onDragOver` prevents default, `onDrop` reads the file, parses, validates, adds.
   - Error display: If validation fails, show error message inline using `aria-live="assertive"`: "This theme file contains unsupported properties and can't be imported. Only standard theme files are accepted." (or the specific error from validateThemeImport)

   **Export section:**
   - "Export Theme" button: Calls `downloadThemeJson(currentTheme, currentOverrides, includeArtwork)`
   - "Copy Share Code" button: Calls `encodeShareCode(currentTheme, currentOverrides)`, copies to clipboard via `navigator.clipboard.writeText()`, shows toast "Copied to clipboard"
   - Share code display: Read-only text field showing the share code (selectable for manual copy)
   - "Include artwork image" checkbox (default: checked)

   **Save as Custom Preset:**
   - "Save as Custom Preset" button
   - Clicking shows an inline text input with placeholder "My Custom Theme" and a confirm button
   - On confirm: Creates a new ThemeDefinition with `id: 'custom-' + crypto.randomUUID()`, `name: userInput`, `builtIn: false`, copies current theme's colors + user overrides merged. Calls `addCustomTheme(newTheme)`.
