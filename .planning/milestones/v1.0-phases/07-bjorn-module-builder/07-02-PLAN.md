    1. Create `src-tauri/src/routes/bjorn.rs` with the following endpoints:

       **GET /api/bjorn/modules** -- List all non-deleted modules for the authenticated user.
       Query: `SELECT id, user_id, name, description, icon, source, config_schema, default_size_w, default_size_h, version, enabled, created_at, updated_at FROM bjorn_modules WHERE user_id = ? AND deleted_at IS NULL ORDER BY updated_at DESC`
       Returns: `Json(json!({ "modules": [...] }))` where each module has fields matching BjornModule type.

       **POST /api/bjorn/modules** -- Create a new module.
       Accepts JSON body: `{ name, description, icon, source, config_schema, default_size_w, default_size_h }`
       Generates UUID id via `util::random_uuid()`. Sets version=1, enabled=1. Inserts into bjorn_modules.
       Also inserts version 1 into bjorn_module_versions.
       Returns: `Json(json!({ "module": { ...created module } }))` with status 201.

       **PUT /api/bjorn/modules/:id** -- Update a module (new version).
       Accepts JSON body: `{ source, config_schema?, name?, description?, icon?, default_size_w?, default_size_h? }`
       Increments version number. Updates the module row. Inserts new version into bjorn_module_versions.
       Prunes versions older than 5 per module: `DELETE FROM bjorn_module_versions WHERE module_id = ? AND version NOT IN (SELECT version FROM bjorn_module_versions WHERE module_id = ? ORDER BY version DESC LIMIT 5)`.
       Returns updated module JSON.

       **DELETE /api/bjorn/modules/:id** -- Soft-delete a module.
       Sets `deleted_at = datetime('now')` and `enabled = 0`. Does NOT hard-delete.
       Returns: `Json(json!({ "ok": true }))`

       **PATCH /api/bjorn/modules/:id/toggle** -- Enable/disable a module.
       Accepts: `{ enabled: bool }`. Updates enabled field.
       Returns updated module.

       **GET /api/bjorn/modules/:id/versions** -- Get version history.
       Query: `SELECT * FROM bjorn_module_versions WHERE module_id = ? ORDER BY version DESC LIMIT 5`
       Returns: `Json(json!({ "versions": [...] }))`

       **POST /api/bjorn/modules/:id/rollback** -- Rollback to a specific version.
       Accepts: `{ version: number }`. Reads source + config_schema from bjorn_module_versions for that version. Updates bjorn_modules with the old source and increments version counter. Inserts new version entry (the rollback is itself a new version).
       Returns updated module.

       **POST /api/bjorn/bridge** -- Data bridge proxy.
       Accepts: `{ source: string, command: string, args?: object }`. Validates source against a simple allowlist of known CLI tools. For now, return `Json(json!({ "error": "Data bridge not yet configured" }))` with a TODO comment -- full implementation depends on tools.json manifest. This is a stub that will be wired in a future iteration.

       All endpoints use `RequireAuth(session)` extractor. All queries bind `session.user_id` for row-level isolation. Use AppError for error responses. Follow the pattern from todos.rs and missions.rs.

    2. Export `pub fn router() -> Router<AppState>` that nests all routes under `/api/bjorn`.
