       - Import configSchema from each of the 11 primitives (StatCard, ProgressGauge, MarkdownDisplay, LineChart, BarChart, ListView, DataTable, FormWidget, KanbanBoard, TimerCountdown, ImageGallery)
       - Export `buildBjornSystemPrompt(): string` that constructs the system prompt per the 07-RESEARCH.md "Bjorn System Prompt Structure" pattern:
         a. Identity: "You are Bjorn, the module builder for OpenClaw Manager."
         b. Task: "Generate a React component that can be rendered as a dashboard widget."
         c. Constraints: MUST use only primitives API, MUST export default function named BjornWidget, MUST accept { widgetId, config, isEditMode, size } props, MUST NOT use dangerous APIs (list the blocklist items), MUST NOT import from external modules, for live data use window.requestData({ source, command })
         d. Primitives API: JSON.stringify each configSchema with its primitive name
         e. WidgetProps interface: `{ widgetId: string, config: Record<string, unknown>, isEditMode: boolean, size: { w: number, h: number } }`
         f. Response format: "Wrap your generated code in a ```javascript code fence. Include ONLY the component code, no imports."
       - Export `extractCodeFromResponse(text: string): string | null` that extracts the first ```javascript or ```jsx or ```tsx code fence from Bjorn's response. Returns null if no code fence found.
       - Export `extractModuleMetadata(text: string): { name: string, description: string }` that attempts to parse a module name and description from Bjorn's conversational text (look for patterns like "Here's a ... module" or "I've created ..."). Falls back to { name: 'Bjorn Module', description: '' }.

    3. Create `frontend/src/pages/chat/BjornTab.tsx`:
       This is the main Bjorn builder UI with chat left + preview right.

       **State management:**
       - `messages` state: array of { id, role, text, timestamp } (same ChatMessage type)
       - `input` state: current input text
       - `sending` state: boolean
       - `generatedSource` state: string (extracted code from latest assistant message)
       - `generationState` state: BjornGenerationState
       - Use `model` from `useLocalStorageState('chat-model', '')`

       **System prompt:**
       - Call `buildBjornSystemPrompt()` once (useMemo) and store as the system prompt context
       - The system prompt is sent as the first message context with each user request

       **Message flow:**
       - User types description and presses send
       - Set generationState to 'generating'
       - Call `api.post('/api/chat', { text: userMessage, model, system_prompt: bjornSystemPrompt })` (same endpoint as regular chat, but with system_prompt field)
       - On response (via polling /api/chat/history or websocket), extract code with `extractCodeFromResponse(lastAssistantMessage.text)`
       - If code extracted: set generatedSource, set generationState to 'analyzing' then 'previewing'
       - If no code: keep generationState at 'idle', display conversational response

       **Approval flow:**
       - onApprove: call `saveBjornModule({ name, description, icon: 'Cube', source: generatedSource, configSchema: { fields: [] }, defaultSize: { w: 3, h: 3 } })`. Use `extractModuleMetadata` for name/description. Set generationState to 'approved'. Add a system message "Module approved and added to dashboard!"
       - onReject: clear generatedSource, set generationState to 'rejected', add system message "Module rejected."
       - onEdit: focus the input, pre-fill with "Please update the module: ", set generationState to 'idle'

       **Layout:**
       ```
       <div style={{ display: 'flex', gap: 16, height: '100%' }}>
         {/* Left: Chat panel */}
         <div style={{ flex: 1, minWidth: 0, display: 'flex', flexDirection: 'column' }}>
           {/* Message list (reuse ChatThread pattern but simplified) */}
           {/* Input bar at bottom */}
         </div>
         {/* Right: Preview panel (only visible when code exists or generating) */}
         {(generatedSource || generationState === 'generating') && (
           <div style={{ flex: 1, minWidth: 0, display: 'flex', flexDirection: 'column' }}>
             <BjornPreview source={generatedSource} generationState={generationState} />
             <BjornApprovalBar onApprove={...} onReject={...} onEdit={...} disabled={sending} generationState={generationState} />
           </div>
         )}
       </div>
       ```

       Use Phosphor icons. Follow project CSS variable conventions for all colors.
