1. In LayoutShell.tsx, import `ScrollRestoration` from `react-router-dom` and add it inside the LayoutShell component (after the closing `</main>` tag, before the Suspense wrappers). ScrollRestoration automatically saves/restores scroll position for the default scroll container on route changes. Since the app uses BrowserRouter (not data routers), we need a manual approach instead.

   **Manual scroll restoration approach** (BrowserRouter does not support ScrollRestoration component):
   - Create a `useScrollRestoration` effect inside LayoutShell that:
     a. Stores scroll position in a module-level Map keyed by pathname when pathname changes (on the PREVIOUS pathname, save `scrollContainerRef.current.scrollTop`)
     b. Restores scroll position from the Map when navigating to a route that has a saved position
   - Add a `ref` to the scrollable wrapper div (the one with `overflowY: 'auto'` on line 341). Name it `scrollContainerRef`.
   - On pathname change, save the PREVIOUS pathname's scroll position and schedule a `requestAnimationFrame` to restore the NEW pathname's scroll position from the map.
   - Use a `useRef` for the previous pathname to detect route changes.

2. Refine the pageEnter animation in globals.css:
   - Change the existing `@keyframes pageEnter` to use opacity only (not translateY) for a cleaner fade:
     ```css
     @keyframes pageEnter {
       from { opacity: 0; }
       to { opacity: 1; }
     }
     ```
   - Reduce duration from 0.25s to 0.15s for snappier feel
   - Update the animation property on the Outlet wrapper div in LayoutShell.tsx to: `animation: 'pageEnter 0.15s ease-out both'`
   - Keep `var(--ease-spring)` only if it matches user intent -- per CONTEXT.md, use CSS opacity+transform fade. Since translateY(4px) is subtle enough and already works, KEEP the translateY but reduce to 2px:
     ```css
     @keyframes pageEnter {
       from { opacity: 0; transform: translateY(2px); }
       to { opacity: 1; transform: translateY(0); }
     }
     ```

3. The scroll position map should be module-level (not state) to persist across re-renders without causing re-renders. Cap the map at 50 entries (delete oldest on overflow) to prevent memory leaks.

IMPORTANT: Do NOT use View Transitions API -- per CONTEXT.md, it crashes WebKitGTK.

1. Extend page-cache.ts with a `usePageState` React hook that:
   - Signature: `usePageState<T>(key: string, initialValue: T): [T, (value: T | ((prev: T) => T)) => void]`
   - Uses the existing in-memory `cache` Map for storage (not localStorage -- form state is ephemeral within a session)
   - On mount, reads from cache and returns cached value if present (within TTL), otherwise returns initialValue
   - On update (via the setter), writes to cache immediately
   - Uses `useSyncExternalStore` pattern for reactivity: add a listener set to page-cache.ts module, notify listeners when cache entries change, so multiple components using the same key stay in sync
   - The hook should be importable as `import { usePageState } from '@/lib/page-cache'`

2. Implementation details:
   - Add `_listeners` Set and `subscribe`/`notifyListeners` functions following the established useSyncExternalStore pattern (same as sidebar-config.ts, keybindings.ts)
   - Add a `getSnapshot` function that returns a stable reference (use a generation counter to trigger re-renders only when data changes)
   - The setter should accept both direct values and updater functions (like useState)
   - Keep existing `getCached` and `setCache` exports unchanged for backward compatibility
   - Add `clearPageCache(key: string)` export for explicit cleanup

3. Create test file `frontend/src/lib/__tests__/page-cache.test.ts` with tests:

   - `setCache` + `getCached` round-trip
   - `clearPageCache` removes entry
   - Cache entries expire after DEFAULT_TTL
