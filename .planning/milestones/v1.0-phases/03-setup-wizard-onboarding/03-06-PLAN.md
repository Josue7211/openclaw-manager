**Create `frontend/src/components/GuidedTour.tsx`:**

- Rendered via `createPortal` into `document.body`
- Only renders when `isTourActive()` returns true

- Backdrop overlay:
  - Full viewport: position fixed, inset 0
  - Background: rgba(0,0,0,0.3)
  - `aria-hidden="true"` (decorative -- tooltip has accessible content)
  - `clip-path: polygon(...)` to create a rectangular cutout around the target element

  Clip-path calculation:
  ```typescript
  // Get target rect via getBoundingClientRect()
  const padding = 8  // px, spotlight padding
  const x1 = rect.left - padding
  const y1 = rect.top - padding
  const x2 = rect.right + padding
  const y2 = rect.bottom + padding

  // Clip-path polygon that covers entire viewport EXCEPT the target rectangle:
  // This uses the evenodd fill rule to create a hole
  const clipPath = `polygon(
    evenodd,
    0 0, 100% 0, 100% 100%, 0 100%, 0 0,
    ${x1}px ${y1}px, ${x1}px ${y2}px, ${x2}px ${y2}px, ${x2}px ${y1}px, ${x1}px ${y1}px
  )`
  ```
  Note: The `evenodd` rule makes the inner polygon a hole in the outer one. Test this approach -- if browser support is an issue, use an SVG mask instead.

- Animation: Backdrop + cutout appearance over 300ms, var(--ease-out). If reduced motion, instant.

- The TourTooltip component is rendered at the calculated position (not inside the backdrop -- it's a sibling element so it's not clipped)

- **Target interaction**: The backdrop has `pointer-events: none` on the cutout area. Or alternatively, the backdrop SVG/clip-path allows clicks through the hole. If using clip-path polygon, the hole naturally allows clicks through since that area is not covered by the element.

- Recalculate spotlight position when:
  - Window resizes (debounced)
  - Step changes
  - Target element might have moved (use ResizeObserver on target if available)

- If the target element for the current stop doesn't exist in the DOM (e.g., user hasn't navigated to that page), show the tooltip at center of screen with a message "Navigate to {section} to see this feature" and auto-advance to next stop after 3 seconds (or let user click Next).

- Tour flow:
  1. Start tour (from wizard "Take a Quick Tour" or demo mode or Settings "Re-run Walkthrough")
  2. Spotlight first stop
  3. User clicks "Next" -> advance to next stop
  4. User clicks "Skip Tour" -> end tour
  5. User clicks "Skip Section" (not shown in MVP, but skipSection() available) -> jump to next section
  6. On last stop, button says "Finish" -> end tour
