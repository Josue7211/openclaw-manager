- Step transition (morphing card):
  - Use state machine: 'idle' | 'exiting' | 'measuring' | 'entering'
  - On step change:
    1. Set state to 'exiting' -- current content applies exit animation (opacity 1->0, scale 1->0.96 forward / 1->1.04 backward, translateY)
    2. After 200ms, render new step offscreen (opacity 0) to measure its height via ref
    3. Animate container height to measured height (250ms, var(--ease-spring))
    4. Set state to 'entering' -- new content applies enter animation (opacity 0->1, scale 1.04->1 forward / 0.96->1 backward)
    5. After 250ms, set state to 'idle'
  - If `shouldReduceMotion()`: skip all transforms, just swap content instantly with optional 50ms opacity crossfade
  - `aria-live="polite"` region announces "Step {N} of 10: {step name}" on step change

- Navigation buttons per UI-SPEC:

  - Back: secondary, hidden on Welcome (step 0)
  - Next: primary, disabled on required steps (1,2,3) until test passes. Label changes: "Get Started" on Welcome, "Next" on middle steps, "Launch Dashboard" on Summary
  - Skip: ghost, visible only on optional steps (4, 5)
  - Take a Quick Tour: secondary, visible only on Summary
