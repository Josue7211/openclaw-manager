# Phase 02.2 Plan 01: Wallbash/Theme Infrastructure Summary

**Rust wallbash color parser + file watcher, dark/light counterpart auto-switch, and WCAG-safe composited surface contrast for all light themes**

## Performance

- **Duration:** 7 min
- **Started:** 2026-03-20T02:42:56Z
- **Completed:** 2026-03-20T02:49:57Z
- **Tasks:** 3
- **Files modified:** 7

## Accomplishments
- Two new Tauri commands (read_wallbash_colors, read_theme_conf) parse HyDE theme files
- notify-based file watcher on ~/.config/hypr/themes/ emits debounced Tauri events on Linux
- COUNTERPART_MAP with 5 bidirectional pairs enables seamless dark<->light theme switching
- setMode() auto-switches to counterpart theme (or default fallback) when toggling modes
- 30 new composited surface WCAG tests catch contrast issues on real visual surfaces
- Catppuccin Latte text colors darkened to pass WCAG AA/AAA on composited bg-panel and bg-card

1. **Task 1: Rust wallbash parser, theme.conf reader, and file watcher** - `9157ab8` (feat)
2. **Task 2: Counterpart map + setMode auto-switch** - `44192c0` (feat)
3. **Task 3: Light theme contrast fix + extended WCAG surface tests** - `a700a98` (fix)

## Files Created/Modified
- `src-tauri/Cargo.toml` - Added notify crate and tempfile dev-dependency
- `src-tauri/src/commands.rs` - Wallbash parser, theme.conf reader, file watcher, 4 Rust tests
- `src-tauri/src/main.rs` - Registered new commands and watcher in invoke_handler + .setup()
- `frontend/src/lib/theme-definitions.ts` - COUNTERPART_MAP, WallbashColors type, Catppuccin Latte text fixes
- `frontend/src/lib/theme-store.ts` - setMode() with counterpart auto-switch logic
- `frontend/src/lib/__tests__/theme-counterpart.test.ts` - 19 tests for counterpart behavior
- `frontend/src/lib/__tests__/theme-contrast.test.ts` - 30 new composited surface WCAG tests

## Decisions Made
- Used `notify` crate v8 with `macos_fsevent` feature for cross-platform file watching
- Created testable inner functions (`read_wallbash_colors_from_path`) separate from Tauri command wrappers to enable unit testing with temp files
- Darkened Catppuccin Latte text-primary from `#4c4f69` to `#44476a` (palette neighbor, 7.4:1 on composited bg-panel)
- Darkened Catppuccin Latte text-secondary from `#65687e` to `#5c5f77` (official Subtext1, 4.9:1 on composited bg-card)
- system->dark/light transitions intentionally do NOT trigger counterpart auto-switch (only explicit dark<->light toggles do)

None - plan executed exactly as written.

## Issues Encountered
- Counterpart test "same mode no-op" initially failed due to module-level state leaking between tests; fixed by using system mode as bridge to avoid triggering counterpart logic when establishing initial dark state.
