### Deviations from Plan (Improvements)

1. **Coalesced wallbash event:** The Rust watcher emits a single `wallbash-theme-update` event with both colors and theme.conf data instead of two separate events. This prevents flash from stale intermediate state during theme switches -- an improvement over the original plan.

2. **Enhanced debounce:** Debounce increased from planned 100ms to 250ms with a 150ms sleep to wait for both files to finish writing, plus event drain. More robust than the original spec.

3. **Wallbash activity detection:** Added `isWallbashActive()` and `wallbashUpdatedRecently()` helpers to prevent polling from interfering with file watcher events -- not in the original plan but prevents race conditions.
