# Requirements: ClawControl

**Defined:** 2026-04-05
**Core Value:** Setup for a self-hosted release should collapse to one Docker Compose stack, one `.env`, one backend URL, and one pairing token so a working package is realistic for non-experts.

## v0.0.8 Requirements

Requirements for the release-package and backend-first setup milestone. Each maps to roadmap phases.

### Release Operations

- [ ] **REL-01**: A release lane exists with explicit acceptance criteria for a "working package"
- [ ] **REL-02**: Branch order, PR ownership, and verification steps are documented before the packaging pass begins

### Backend Stack

- [ ] **STACK-01**: A single Docker Compose deployment can boot the backend stack needed for desktop setup and core flows
- [ ] **STACK-02**: Users configure the stack through one `.env` file instead of per-service manual edits
- [ ] **STACK-03**: Service-specific secrets stay scoped to the services that consume them instead of leaking into unrelated app config

### Setup Contract

- [ ] **SETUP-01**: The backend exposes a setup/status contract that reports health, missing configuration, and enabled capabilities
- [ ] **SETUP-02**: The backend can issue or validate a pairing token for the desktop app
- [ ] **SETUP-03**: The desktop app can complete initial setup from backend URL + pairing token without walking users through each backend service manually

### Package Hardening

- [ ] **PKG-01**: The packaged app can sign in, reach the backend, and load core chat/dashboard flows against the backend-first setup
- [ ] **PKG-02**: When optional integrations are offline, the packaged app shows clean degraded states instead of hanging or blank pages
- [ ] **PKG-03**: Release documentation explains how to boot the backend stack and connect the desktop package with minimal steps

## Out of Scope

| Feature | Reason |
|---------|--------|
| Replacing Tauri with Electron | Does not solve the current release/setup bottleneck |
| Containerizing Apple-device bridges | Those can remain external while the core release path is simplified |
| AgentShell / MemD feature expansion | Deferred until the release path is stable and packageable |
